单调栈

  单调栈即满足单调性的栈结构。与单调队列相比，其只在一端进行进出。借助单调性处理问题的思想在于及时排除不可能的选项，保持策略集合的高度有效性和秩序性。为了描述方便，以下以维护一个整数单调递增的栈为例。

  将一个元素插入单调栈时，为了维护栈的单调性，需要在保证将该元素插入到栈顶后，整个栈满足单调性的前提下弹出最少的元素。

  例如，栈中自顶向下的元素为 $1,3,5,10,30,50$，插入元素 $20$ 时，为了保证单调性，需要依次弹出元素 $1,3,5,10$，操作后栈变为 $20,30,50$。

cin>>x;
while(!sta.empty()&&sta.top()<x)
    sta.pop();
sta.push(x);

例题

P5788 【模板】单调栈

  给出长为 $n$ 的整数序列 $a$。定义函数 $f(i)$ 为序列中第 $i$ 个元素之后第一个大于 $a_i$ 的元素的下标，即 $f(i)=\min\limits_{i<j\leq n, a_i < a_j} {j}$。若不存在，则 $f(i)=0$。

  求 $f(i)(1\leq i\leq n)$。

  数据范围：$1 \le n\leq 3\times 10^6,1\leq a_i\leq 10^9$。

分析

  从后往前遍历，对于每个元素，弹出栈顶比它小的元素，此时栈顶就是答案，之后加入该元素。注意栈中的元素都是序列的下标。

代码

#include<bits/stdc++.h>
using namespace std;
stack<int> sta;
int a[3000010],f[3000010];
int main()
{
    long long n,ans=0;
    cin>>n;
    for(int i=1;i<=n;i++)
        scanf("%d",&a[i]);
    for(int i=n;i>=1;i--)
    {
        while(!sta.empty()&&a[sta.top()]<=a[i])
            sta.pop();
        if(sta.empty())
            f[i]=0;
        else
            f[i]=sta.top();
        sta.push(i);
    }
    for(int i=1;i<=n;i++)
        printf("%d ",f[i]);
    cout<<endl;
    return 0;
}

POJ-2559 Largest Rectangle in a Histogram

  如下图所示，在一条水平线上方有若干个矩形，求包含于这些矩形的并集内部的最大矩形的面积（在下图中，答案就是阴影部分的面积），矩形个数 $\leq 10^5$。

分析

  我们先来思考这样一个问题：如果矩形的高度从左到右单调递增，那么答案是多少？显而易见，我们可以尝试以每个矩形的高度作为最终矩形的高度，并把宽度延伸到右边界，得到一个矩形，在所有这样的矩形面积中取最大值就是答案。如下图所示：

  如果下一个矩形的高度比上一个小，那么该矩形想利用之前的矩形一起构成一块较大的面积时，这块面积的高度就不可能超过该矩形自己的高度。换句话说，在考虑完上图的四种情况后，下图中打叉的那部分形状就没有丝毫用处了。

  既然没有用处，为什么不把这些比该矩形高的矩形都删掉，用一个宽度累加、高度为该矩形自己的高度的新矩形（就是上图中的阴影部分）代替呢？这样并不会对后续的计算产生影响。于是我们维护的轮廓就变成了一个高度始终单调递增的矩形序列，问题变得可解了。

  详细地说，我们建立一个栈，用来保存若干个矩形，这些矩形的高度是单调递增的。我们从左到右依次扫描每个矩形：

  如果当前矩形比栈顶矩形高，直接进栈。

  否则不断取出栈顶，直至栈为空或者栈顶矩形的高度比当前矩形小。在出栈过程中，我们累计被弹出的矩形的宽度之和，并且每弹出一个矩形，就用它的高度乘上累计的宽度去更新答案。整个出栈过程结束后，我们把一个高度为当前高度、宽度为累计值的新矩形入栈。

  整个扫描结束后，我们把栈中剩余的矩形依次弹出，按照与上面相同的方法更新答案。为了简化程序实现，也可以增加一个高度为 $0$ 的矩形 $a[n+1]$，以避免在扫描结束后栈中有剩余矩形。

代码

#include<iostream>
#include<cstdio>
using namespace std;
int st[100010],a[100010],w[100010],top;
long long ans;
int main()
{
    int n;
    while(cin>>n&&n)
    {
        ans=0;
        a[n+1]=top=0;
        for(int i=1;i<=n+1;i++)
        {
            if(i!=n+1)
                scanf("%d",&a[i]);
            if(a[i]>st[top])
                st[++top]=a[i],w[top]=1;
            else
            {
                int width=0;
                while(st[top]>a[i])
                {
                    width=width+w[top];
                    ans=max(ans,(long long)width*st[top]);
                    top--;
                }
                st[++top]=a[i];
                w[top]=width+1;
            }
        }
        cout<<ans<<endl;
    }
    return 0;
}

单调队列

  单调队列的重点分为 “单调” 和 “队列”。”单调” 指的是元素的 “规律”——递增或递减，”队列” 指的是元素只能从队头和队尾进行操作。因为每个元素至多入队一次、出队一次，所以整个算法的时间复杂度为 $O(N)$。它的思想是在决策集合（队列）中及时排除一定不是最优解的选择。单调队列也是优化动态规划的一个重要手段。

例题

P1886 滑动窗口/【模板】单调队列

题目描述

  有一个长为 $n$ 的序列 $a$，以及一个大小为 $k$ 的窗口。现在这个从左边开始向右滑动，每次滑动一个单位，求出每次滑动后窗口中的最大值和最小值。

  例如：The array is $[1,3,-1,-3,5,3,6,7]$， and $k=3$。

输入格式

  输入一共有两行，第一行有两个正整数 $n,k(1\leq k\leq n\leq 10^6)$。

  第二行 $n$ 个整数，表示序列 $a,a_i\in[-2^{31},2^{31})$。

输出格式

  输出共两行，第一行为每次窗口滑动的最小值，第二行为每次窗口滑动的最大值。

分析

  暴力的方法很简单，对于每一段 $i$ ~ $i+k-1$ 的序列，逐个比较来找出最大值和最小值，时间复杂度为 $O(nk)$。

  显然，这其中进行了大量重复工作，除了开头 $k-1$ 个和结尾 $k-1$ 个数之外，每个数都进行了 $k$ 次比较，而题中 $100%$ 的数据为 $n\leq 1000000$，当 $k$ 稍大的情况下，显然会超时。

  要求的是每连续的 $k$ 个数中的最大（最小）值，很明显，当一个数进入所要 “寻找” 最大值的范围中时，若这个数比其前面（先进队）的数要大，显然，前面的数会比这个数先出队且不再可能是最大值。

  也就是说，当满足以上条件时，可将前面的数字 “弹出”，再将该数 $\text{push}$ 进队尾。

  这就相当于维护了一个递减的队列，符合单调队列的定义，减少了重复的比较次数，不仅如此，由于维护的队伍是查询范围内的且是递减的，队头必定是该查询区域内的最大值，因此输出时只需输出队头即可。

  由于查询区间的长度是固定的，超出查询空间的值再大也不能输出，因此还需要 $p$ 数组记录第 $i$ 个队列中的数字在原数组中的位置，以弹出越界的队首。

  每次取最小值的算法流程：

代码

#include<bits/stdc++.h>
using namespace std;
int a[1000010],q[1000010],p[1000010];
int main()
{
    int n,k;
    cin>>n>>k;
    for(int i=1;i<=n;i++)
        scanf("%d",&a[i]);
    int    head=1,tail=0;
    for(int i=1;i<=n;i++)
    {
        while(head<=tail&&q[tail]>=a[i])//要队列里有元素，并且队尾元素比当前元素大，表示队尾元素不可能作为最小值，所以从队尾出队。直到队尾元素小于当前值，满足单调。 
            tail--;
        q[++tail]=a[i];//待处理值入队 
        p[tail]=i;//存下当前处理值的编号 
        while(p[head]<=i-k)
            head++;//如果队首元素已经不在范围内，从队首出队
        if(i>=k)
            printf("%d ",q[head]); 
    } 
    cout<<endl;
    memset(q,0,sizeof(q));
    memset(p,0,sizeof(p));
    head=1,tail=0;
    for(int i=1;i<=n;i++)//a[i]为当前要处理的值 
    {
        while(head<=tail&&q[tail]<=a[i])//只要队列里有元素，并且队尾元素比当前元素小，表示队尾元素不可能作为最大值，所以从队尾出队。直到队尾元素大于当前值，满足单调。 
            tail--;
        q[++tail]=a[i];//待处理值入队 
        p[tail]=i;//存下当前处理值的编号 
        while(p[head]<=i-k)
            head++;//如果队首元素已经不在范围内，从队首出队 
        if(i>=k)
            printf("%d ",q[head]); 
    }
    cout<<endl;
    return 0;
}

最大子序和

https://ac.nowcoder.com/acm/contest/1006/D

  给定一个长度为 $N$ 的整数序列（可能有负数），从中找出一段长度不超过 $M$ 的连续子区间，使得子区间中所有数的和最大$(N,M\leq 2\ast 10^5)$。

分析

  计算 “区间和” 的问题，一般转化为 “两个前缀和相减” 的形式进行求解。我们先求出 $S[i]$ 表示序列里前 $i$ 项的和，则连续子区间 $[L,R]$ 中数的和就等于 $S[R]-S[L-1]$。所以原问题可以转化为：找出两个位置 $x,y$，使 $S[y]-S[x]$ 最大并且 $y-x\leq M$。

  首先我们枚举右端点 $i$，当 $i$ 固定时，问题就变成为：找到一个左端点 $j$，其中 $j\in[i-m,i-1]$ 并且 $S[j]$ 最小。

  不妨比较一下任意两个位置 $j$ 和 $k$，如果 $k<j<i$ 并且 $S[k]\geq S[j]$，那么对于所有大于等于 $i$ 的右端点，$k$ 永远不会成为最优选择。这是因为不但 $S[k]$ 不小于 $S[j]$，而且 $j$ 离 $i$ 更近，长度更不容易超过 $M$，即 $j$ 的生存能力比 $k$ 更强。所以当 $j$ 出现后，$k$ 就是完全是一个无用的位置。

  以上比较告诉我们，可能成为最优选择的策略集合一定是一个 “下标位置递增、对应的前缀和 $S$ 的值也递增” 的序列。我们可以用一个队列保存这个队列。随着右端点变从前向后扫描，我们对每个 $i$ 执行以下三个步骤：

  $1.$判断队头决策与 $i$ 的距离是否超出 $M$ 的范围，若超过则出队。

  $2.$此时队头就是右端点为 $i$ 时，左端点 $j$ 的最优选择。

  $3.$不断删除队尾决策，直到队尾对应的 $S$ 值小于 $S[i]$。然后把 $i$ 作为一个新的决策入队。

代码

#include<bits/stdc++.h>
using namespace std;
int a[300010],q[300010],sum[300010];
int main()
{
    int n,m;
    cin>>n>>m;
    for(int i=1;i<=n;i++)
    {
        scanf("%d",&a[i]);
        sum[i]=sum[i-1]+a[i];
    }
    int head=1,tail=1,ans=-0x3f3f3f3f;
    q[1]=0;
    for(int i=1;i<=n;i++)
    {
        while(head<=tail&&q[head]<i-m)
            head++;
        ans=max(ans,sum[i]-sum[q[head]]);
        while(head<=tail&&sum[q[tail]]>=sum[i])
            tail--;
        q[++tail]=i;
    }
    cout<<ans<<endl;
    return 0;
}

HDU-3415 Max Sum of Max-K-sub-sequence

  给定一个环状序列 $a_1,a_2,\cdots,a_n$，其中 $a_1$ 的左边是 $a_n$。求最大的连续和，长度不超过 $k(1\leq n\leq 10^5,1\leq k\leq n,-1000\leq a_i\leq 1000)$。

  输出满足条件的最大连续和以及起始位置和终止位置。如果有多个结果，输出起始位置下标最小的，如果还是有多个结果，输出长度最短的。

分析

  把原序列复制一倍添加到最后，这样把环形序列处理成了长度为原来 $2$ 倍的普通序列。之后用单调队列维护在 $k$ 个数之内的最小值的下标即可。

代码

#include<bits/stdc++.h>
using namespace std;
int a[200010],q[200010],sum[200010];
int main()
{
    int T;
    cin>>T;
    while(T--)
    {
        int n,k;
        scanf("%d%d",&n,&k);
        for(int i=1;i<=n;i++)
        {
            scanf("%d",&a[i]);
            a[i+n]=a[i];
        }
        memset(sum,0,sizeof(sum));
        for(int i=1;i<=2*n;i++)
            sum[i]=sum[i-1]+a[i];
        int head=0,tail=0,l,r,maxn=-0x3f3f3f3f;
        memset(q,0,sizeof(q));
        q[0]=0;
        for(int i=1;i<=2*n;i++)
        {
            while(head<=tail&&q[head]<i-k)
                head++;
            if(sum[i]-sum[q[head]]>maxn)
            {
                maxn=sum[i]-sum[q[head]];
                l=q[head]+1;
                r=i;
            }
            while(head<=tail&&sum[q[tail]]>sum[i])
                tail--;
            q[++tail]=i;
        }
        if(l>n)    l=l-n;
        if(r>n)    r=r-n;
        printf("%d %d %d\n",maxn,l,r);
    }
    return 0;
}
