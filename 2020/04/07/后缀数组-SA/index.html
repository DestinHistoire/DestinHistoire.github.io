后缀数组(SA)
字符串 后缀数组
算法笔记
发布日期:   2020-04-07

概述

  本文中，字符串下标从 $1$ 开始，”后缀 $i$” 代指以第 $i$ 个字符开头的后缀。

  后缀数组（$\text{Suffix Array}$）主要是两个数组：$sa$ 和 $rk$。其中，$sa[i]$ 表示将所有后缀排序后字典序第 $i$ 小的后缀的下标，$rk[i]$ 表示后缀 $i$ 的排名。

  这两个数组满足性质：$sa[rk[i]]=rk[sa[i]]=i$。

后缀数组计算方法

$O(n^2\log n)$ 做法

  $\text{string+sort}$，比较两个字符串是 $O(n)$ 的，排序是 $ O(n\log n) $ 的。

$O(n\log^2 n)$ 做法

  这个做法用到了倍增的思想。

  先对每个长度为 $1$ 的子串（即每个字符）进行排序。

  假设我们已经知道了长度为 $w$ 的子串的排名 $rk_n[1\cdots n]$（即 $rk_w[i]$ 表示 $s[i\cdots\min(i+w-1,n)]$ 在 ${s[x\cdots\min(x+w-1,n)]|x\in [1,n]}$ 中的排名），那么，以 $rk_w[i]$ 为第一关键字，以 $rk_w[i+w]$ 为第二关键字（若 $i+w>n$，则令 $rk_w[i+w]$ 为 $0$）进行排序，就可以求出 $rk_{2w}[1\cdots n]$。

  倍增排序示意图：

  如果用 $\text{sort}$ 进行排序，复杂度就是 $O(n\log^2n)$ 的。

代码

#include <bits/stdc++.h>
using namespace std;
const int N=1000010;
char s[N];
int w,sa[N],rk[N<<1],oldrk[N<<1];//为了防止rk[i+w]导致数组越界，开两倍数组 
bool cmp(int x,int y)
{
    if(rk[x]==rk[y])
        return rk[x+w]<rk[y+w];
    return rk[x]<rk[y];
}
int main()
{
    scanf("%s",s+1);
    int n=strlen(s+1);
    for(int i=1;i<=n;i++)
        rk[i]=s[i];
    for(w=1;w<n;w<<=1)
    {
        for(int i=1;i<=n;i++)
            sa[i]=i;
        sort(sa+1,sa+1+n,cmp);
        memcpy(oldrk,rk,sizeof(rk));//由于计算rk的时候原来的rk会被覆盖，需要先复制一份 
        for(int i=1,p=0;i<=n;i++)
        {
            if(oldrk[sa[i]]==oldrk[sa[i-1]]&&oldrk[sa[i]+w]==oldrk[sa[i-1]+w])
                rk[sa[i]]=p;//若两个子串相同，它们对应的rk也需要相同，所以要去重 
            else
                rk[sa[i]]=++p; 
        }
    }
    for(int i=1;i<=n;i++)
        printf("%d ",sa[i]);
    cout<<endl; 
    return 0;
}

$O(n\log n)$ 做法

  在刚刚的 $O(n\log^2 n)$ 做法中，单次排序是 $O(n\log n)$ 的，如果能 $O(n)$ 排序，就能 $O(n\log n)$ 计算后缀数组了。

  由于计算后缀数组的过程中排序的关键字是排名，值域为 $O(n)$，并且是一个双关键字的排序，可以使用基数排序优化至 $O(n)$。

计数排序

  计数排序是一种复杂度为 $O(n+w)$ 的稳定排序，其中 $w$ 代表待排序数据的值域大小。

  计数排序分为三个步骤：

  $1.$ 计算每个数出现了几次。

  $2.$ 求出每个数出现次数的前缀和。

  $3.$ 利用出现次数的前缀和，从右至左计算每个数的排名。

参考代码

const int N=100010,W=100010;
int n,w,a[N],cnt[W],b[N];
void counting_sort() 
{
    memset(cnt,0,sizeof(cnt));
    for(int i=1;i<=n;i++) 
        cnt[a[i]]++;
    for(int i=1;i<=w;i++) 
        cnt[i]=cnt[i]+cnt[i-1];
    for(int i=n;i>=1;--i) 
        b[cnt[a[i]]--]=a[i];
}

基数排序

  基数排序是将待排序的元素拆分为 $k$ 个关键字（比较两个元素时，先比较第一关键字，如果相同再比较第二关键字……），然后先对第 $k$ 关键字进行稳定排序，再对第 $k-1$ 关键字进行稳定排序，再对第 $k-2$ 关键字进行稳定排序……最后对第一关键字进行稳定排序，这样就完成了对整个待排序序列的稳定排序。

  一般来说，每个关键字的值域都不大，就可以使用计数排序作为内层排序，复杂度为 $O(nk+\sum\limits_{i=1}^{k}w_i)$，其中 $w_i$ 为第 $i$ 关键字的值域大小（如果关键字值域很大，就可以直接使用基于比较的 $O(nk\log n)$ 排序而无需使用基数排序了）。

代码

const int N=100010;
const int W=100010;
const int K=100;
int n,w[K],k,cnt[W];
struct element
{
    int key[K];
    bool operator<(const element &y) const
    {
        for(int i=1;i<=k;i++)
        {
            if(key[i]==y.key[i])
                continue;
            return key[i]<y.key[i];
        }
        return false;    
    }
}a[N],b[N];
void counting_sort(int p)
{
    memset(cnt,0,sizeof(cnt));
    for(int i=1;i<=n;i++)
        cnt[a[i].key[p]]++;
    for(int i=1;i<=w[p];i++)
        cnt[i]=cnt[i]+cnt[i-1];
    for(int i=n;i>=1;i--)
        b[cnt[a[i].key[p]]--]=a[i];
    memcpy(a,b,sizeof(a));    
}
void radix_sort()
{
    for(int i=k;i>=1;i--)
        counting_sort(i);
}

常数优化

第二关键字无需计数排序

  先把 $s[i+w\cdots i+2w-1]$ 为空串（即第二关键字为 $0$）的位置放前面，再把剩下的按排好的顺序放进去。

for(int i=n,p=0;i>n-w;i--)
    ID[++p]=i;
for(int i=1;i<=n;i++)
    if(sa[i]>w)
        ID[++p]=sa[i]-w;

优化计数排序的值域

  每次对 $rk$ 进行去重之后，我们都计算了一个 $p$，这个 $p$ 即是 $rk$ 的值域，将值域改成它即可。

将 $rk[ID[i]]$ 存下来，减少不连续内存访问

  这个优化在数据范围较大时效果非常明显。

用函数 $\text{cmp}$ 来计算是否重复

  同样是减少不连续内存访问，在数据范围较大时效果比较明显。

  把 

oldrk[sa[i]]==oldrk[sa[i-1]]&&oldrk[sa[i]+w]==oldrk[sa[i-1]+w]

  替换成

cmp(sa[i],sa[i-1],w);
bool cmp(int x,int y,int w)
{
    return oldrk[x]==oldrk[y]&&oldrk[x+w]==oldrk[y+w];
}

【例题】P3809 【模板】后缀排序

  给定长度为 $n(1\leq n\leq 10^6)$ 的字符串，按顺序输出后缀的第一个字符在原串中的位置。位置编号为 $1$ 到 $n$。

参考代码

#include<bits/stdc++.h>
using namespace std;
const int N=1000010;
char s[N];
int n,m,w,sa[N],rk[N<<1],oldrk[N<<1],ID[N],px[N],cnt[N];//px[i]=rk[id[i]],用于排序的数组
bool cmp(int x,int y,int w) 
{    
    return oldrk[x]==oldrk[y]&&oldrk[x+w]==oldrk[y+w];
}
int main() 
{
    scanf("%s",s+1);
    n=strlen(s+1);
    m=300;
    for(int i=1;i<=n;++i)
    {
        rk[i]=s[i];
        cnt[rk[i]]++;
    }
    for(int i=1;i<=m;++i) 
        cnt[i]=cnt[i]+cnt[i-1];
    for(int i=n;i>=1;--i)
    {
        sa[cnt[rk[i]]]=i;
        cnt[rk[i]]--;
    }
    int p;
    for(w=1;w<n;w<<=1,m=p)//m=p就是优化计数排序值域 
    {
        p=0;
        for(int i=n;i>n-w;--i) 
            ID[++p]=i;
        for(int i=1;i<=n;++i)
            if(sa[i]>w) 
                ID[++p]=sa[i]-w;
        memset(cnt,0,sizeof(cnt));
        for(int i=1;i<=n;++i)
        {
            px[i]=rk[ID[i]];
            cnt[px[i]]++;
        }
        for(int i=1;i<=m;++i) 
            cnt[i]=cnt[i]+cnt[i-1];
        for(int i=n;i>=1;--i) 
        {    
            sa[cnt[px[i]]]=ID[i];

  喜欢钻研问题的 JS 同学，最近又迷上了对加密方法的思考。一天，他突然想出了一种他认为是终极的加密办法：把需要加密的信息排成一圈，显然，它们有很多种不同的读法。

从字符串首尾取字符最小化字典序

【例题】P2870 [USACO07DEC]Best Cow Line G

  给定一个字符串 $S(1\leq|S|\leq 5\times 10^5)$，每次从首或尾取一个字符组成字符串，问所有能够组成的字符串中字典序最小的一个。

分析

  暴力做法就是每次最坏 $O(n)$ 地判断当前应该取首还是尾（即比较取首得到的字符串与取尾得到的反串的大小），只需优化这一判断过程即可。

  由于需要在原串后缀与反串后缀构成的集合内比较大小，可以将反串拼接在原串后，并在中间加上一个没出现过的字符（如 #，代码中可以直接使用空字符），求后缀数组，即可 $O(n)$ 完成这一判断。

代码

#include<bits/stdc++.h>
using namespace std;
const int N=1000010;
char s[N];
int n,m,w,sa[N],rk[N<<1],oldrk[N<<1],ID[N],px[N],cnt[N];
bool cmp(int x,int y,int w)
{
    return oldrk[x]==oldrk[y]&&oldrk[x+w]==oldrk[y+w];
}
int main()
{
    cin>>n;
    int l=1,r=n,tot=0;
    m=200;
    for(int i=1;i<=n;++i)
        while(!isalpha(s[i]=getchar()));
    for(int i=1;i<=n;++i)
    {
        rk[i]=s[i];
        rk[2*n-i+2]=s[i]; 
    }
    n=2*n+1;
    for(int i=1;i<=n;i++)
        cnt[rk[i]]++;
    for(int i=1;i<=m;++i)
        cnt[i]=cnt[i]+cnt[i-1];
    for(int i=n;i>=1;--i)
    {
        sa[cnt[rk[i]]]=i;
        cnt[rk[i]]--;
    }
    int p;
    for(w=1;w<n;w<<=1,m=p)
    {
        p=0;
        for(int i=n;i>n-w;--i)
            ID[++p]=i;
        for(int i=1;i<=n;++i)
            if(sa[i]>w)
                ID[++p]=sa[i]-w;
        memset(cnt,0,sizeof(cnt));
        for(int i=1;i<=n;++i)
        {
            px[i]=rk[ID[i]];
            cnt[px[i]]++;
        }
        for(int i=1;i<=m;++i)
            cnt[i]=cnt[i]+cnt[i-1];
        for(int i=n;i>=1;--i)
        {
            sa[cnt[px[i]]]=ID[i];
            cnt[px[i]]--;
        }
        memcpy(oldrk,rk,sizeof(rk));
        for(int i=1,p=0;i<=n;++i)
        {
            if(cmp(sa[i],sa[i-1],w))
                rk[sa[i]]=p;
            else
                rk[sa[i]]=++p;
        }
    }
    while(l<=r)
    {
        //printf("l=%d,n+1-r=%d\n",l,n+1-r);
        //printf("rk[l]=%d,rk[n+1-r]=%d\n",rk[l],rk[n+1-r]);
        if(rk[l]<rk[n+1-r])
        {
            printf("%c",s[l]);
            l++;
        }
        else
        {
            printf("%c",s[r]);
            r--;
        }
        tot++;
        if(tot%80==0)
            puts("");
    }
    return 0;
}

$\text{height}$ 数组

$\text{LCP}$（最长公共前缀）

  两个字符串 $S$ 和 $T$ 的 $\text{LCP}$ 就是最大的 $x(x\leq \min(|S|,|T|))$ 使得 $S_i=T_i(\forall 1\leq i\leq x)$。

  下文中以 $\text{LCP}(i,j)$ 表示后缀 $i$ 和后缀 $j$ 的最长公共前缀（的长度）。

$\text{height}$ 数组的定义

  $\text{height}[i]=\text{LCP}(sa[i],sa[i-1])$，即第 $i$ 名的后缀与第 $i-1$ 名的后缀的最长公共前缀。$\text{height}[1]$ 可以视为 $0$。

$O(n)$ 求 $\text{height}$ 数组需要的引理

$$
\text{height}[rk[i]]\geq \text{height}[rk[i-1]]-1
$$

证明：

  当 $\text{height}[rk[i-1]]\leq 1$ 时，上式显然成立（右边小于等于 $0$）。

  当 $\text{height}[rk[i-1]]>1$ 时：

  设后缀 $i-1$ 为 $aAD$（$A$ 是一个长度为 $\text{height}[rk[i-1]]-1$ 的字符串），那么后缀 $i$ 就是 $AD$。设后缀 $sa[rk[i-1]-1]$ 为 $aAB$，那么 $\text{LCP}(i-1,sa[rk[i-1]-1])=aA$。由于后缀 $sa[rk[i-1]-1]+1$ 是 $AB$，一定排在后缀 $i$ 的前面，所以后缀 $sa[rk[i]-1]$ 一定含有前缀 $i$，所以 $\text{LCP}(i,sa[rk[i]-1])$ 至少是 $\text{height}[rk[i-1]]-1$。

  简单来说：

  $i-1$：$aAD$

  $i$：$AD$

  $sa[rk[i-1]-1]$：$aAB$

  $sa[rk[i-1]-1]+1$：$AB$

  $sa[rk[i]-1]$：$A[B/C]$

  $\text{LCP}(i,sa[rk[i]-1]) $：$AX$（$X$ 可能为空）

$O(n)$ 求 $\text{height}$ 数组的代码实现

  利用上面的引理暴力求即可。

for(int i=1,k=0;i<=n;i++)
{
    if(k!=0)
        k--;
    while(s[i+k]==s[sa[rk[i]-1]+k])
        k++;
    ht[rk[i]]=k;
}

  $k$ 不会超过 $n$，最多减 $n$ 次，所以最多加 $2n$ 次，总时间复杂度为 $O(n)$。

$\text{height}$ 数组的应用

两子串最长公共前缀

  性质 $1$：$\forall i\leq k\leq j,\text{LCP}(i,j)=\min(\text{LCP}(i,k),\text{LCP}(k,j))$。

  性质 $2$：$\forall 1<k\leq j,\text{LCP}(i,j)=\min{\text{LCP}(k-1,k)}$。

  性质 $1$ 显然正确，它的意义在于用来证明性质 $2$。性质 $2$ 的意义在于提供了一个将 $\text{LCP}$ 问题转化为 $\text{RMQ}$ 问题的方法：
$$
\text{LCP}(sa[i],sa[j])=\min{\text{height}[i+1]\cdots \text{height}[j]}
$$
  使用 $\text{RMQ}$ 算法即可，预处理的时间复杂度为 $O(n\log n)$，查询的时间复杂度为 $O(1)$。

【例题】UVA - 12338 Anti-Rhyme Pairs

  $T$ 组数据，每组数据给定 $N$ 个字符串，$q$ 次询问，每次询问求出编号为 $l,r$ 的字符串的最长公共前缀的长度。

  数据范围：$T\leq 35,N\leq 10^5,\sum|S|\leq 10^6,1\leq q\leq 10^6$。

分析

  将 $n$ 个字符串拼接起来，中间用不相同且没有出现在字符串中的字符隔开，求出后缀数组，使用 $\text{ST}$ 表维护 $\text{height}$ 数组，每次查询计算 $\text{query}(rk[pos[l]]+1,rk[pos[r]])$ 即可，设拼接后的字符串的长度为 $L=\sum\limits_{i=1}^{n}|s_i|$，时间复杂度 $O\Big(T\times\big(L\log L+q\big)\Big)$。

代码

比较一个字符串的两个子串的大小关系

  假设需要比较的是 $A=S[a\cdots b]$ 和 $B=S[c\cdots d]$ 的大小关系。

  若 $\text{LCP}(a,c)\geq \min(|A|,|B|)$，则 $A<B\Longleftrightarrow |A|\leq |B|$

  否则，$A\leq B\Longleftrightarrow rk[a]<rk[b]$。

不同子串的数目

  子串就是后缀的前缀，所以可以枚举每个后缀，计算前缀总数，再减掉重复的。

  ”前缀总数”其实就是子串个数，为 $\frac{n(n+1)}{2}$。

  如果按后缀排序的顺序枚举后缀，每次新增的子串就是除了与上一个后缀的 $\text{LCP}$ 之外剩下的前缀。这些前缀一定是新增的，否则会破坏 $\text{LCP}(sa[i],sa[j])=\min{\text{height}[i+1\cdots j]}$。只有这些前缀是新增的，因为 $\text{LCP}$ 部分在枚举上一个前缀时已经计算过了。

  所以答案为：
$$
\frac{n(n+1)}{2}-\sum\limits_{i=2}^{n}\text{height}[i]
$$

【例题】P2408 不同子串个数

  给定一个长为 $n(1\leq n\leq 10^5)$ 的字符串，求本质不同的子串的个数。

代码

#include<bits/stdc++.h>
using namespace std;
const int N=100010;
char s[N];
int n,m,w,sa[N],rk[N<<1],oldrk[N<<1],ID[N],px[N],cnt[N],height[N];
bool cmp(int x,int y,int w)
{
    return oldrk[x]==oldrk[y]&&oldrk[x+w]==oldrk[y+w];
}
void SA(int n,int m)
{
    for(int i=1;i<=n;++i)
    {
        rk[i]=s[i];
        cnt[rk[i]]++;
    }
    for(int i=1;i<=m;++i) 
        cnt[i]=cnt[i]+cnt[i-1];
    for(int i=n;i>=1;--i)
    {
        sa[cnt[rk[i]]]=i;
        cnt[rk[i]]--;
    }
    int p;
    for(w=1;w<n;w<<=1,m=p)//m=p就是优化计数排序值域 
    {
        p=0;
        for(int i=n;i>n-w;--i) 
            ID[++p]=i;
        for(int i=1;i<=n;++i)
            if(sa[i]>w) 
                ID[++p]=sa[i]-w;
        memset(cnt,0,sizeof(cnt));
        for(int i=1;i<=n;++i)
        {
            px[i]=rk[ID[i]];
            cnt[px[i]]++;
        }
        for(int i=1;i<=m;++i) 
            cnt[i]=cnt[i]+cnt[i-1];
        for(int i=n;i>=1;--i) 
        {    
            sa[cnt[px[i]]]=ID[i];
            cnt[px[i]]--;
        }
        memcpy(oldrk,rk,sizeof(rk));
        for(int i=1,p=0;i<=n;++i)
        {
            if(cmp(sa[i],sa[i-1],w))
                rk[sa[i]]=p;
            else
                rk[sa[i]]=++p;
        }
    }
} 
void get_height(int n)
{
    for(int i=1,k=0;i<=n;i++)
    {
        if(k!=0)
            k--;
        while(s[i+k]==s[sa[rk[i]-1]+k])
            k++;
        height[rk[i]]=k;
    }
}
int main()
{
    cin>>n;
    scanf("%s",s+1);
    m=300;
    SA(n,m);
    get_height(n);
    long long ans=1ll*n*(n+1)/2;
    for(int i=1;i<=n;i++)
        ans=ans-height[i];
    cout<<ans<<endl;
    return 0;
}

出现至少 $k$ 次的子串的最大长度

【例题】P2852 [USACO06DEC]Milk Patterns G

  给定序列 $s(1\leq n\leq 20000,0\leq s_i\leq 10^6)$ 求最长的至少重复 $k(2\leq k\leq n)$ 次的子串。

分析

  出现至少 $k$ 次意味着后缀排序后有至少连续 $k$ 个后缀的 $\text{LCP}$ 是这个子串。所以，求出每相邻 $k-1 $ 个 $\text{height}$ 的最小值，再求这些最小值的最大值就是答案。

  使用单调队列维护 $\text{height}$ 数组，时间复杂度为 $O(n)$。

代码

#include<bits/stdc++.h>
using namespace std;
const int N=100010;
int s[N];
int n,m,w,k,sa[N],rk[N<<1],oldrk[N<<1],ID[N],px[N],cnt[1000010],height[N],q[N];
bool cmp(int x,int y,int w)
{
    return oldrk[x]==oldrk[y]&&oldrk[x+w]==oldrk[y+w];
}
void SA(int n,int m)
{
    for(int i=1;i<=n;++i)
    {
        rk[i]=s[i];
        cnt[rk[i]]++;
    }
    for(int i=1;i<=m;++i) 
        cnt[i]=cnt[i]+cnt[i-1];
    for(int i=n;i>=1;--i)
    {
        sa[cnt[rk[i]]]=i;
        cnt[rk[i]]--;
    }
    int p;
    for(w=1;w<n;w<<=1,m=p)//m=p就是优化计数排序值域 
    {
        p=0;
        for(int i=n;i>n-w;--i) 
            ID[++p]=i;
        for(int i=1;i<=n;++i)
            if(sa[i]>w) 
                ID[++p]=sa[i]-w;
        memset(cnt,0,sizeof(cnt));
        for(int i=1;i<=n;++i)
        {
            px[i]=rk[ID[i]];
            cnt[px[i]]++;
        }
        for(int i=1;i<=m;++i) 
            cnt[i]=cnt[i]+cnt[i-1];
        for(int i=n;i>=1;--i) 
        {    
            sa[cnt[px[i]]]=ID[i];
            cnt[px[i]]--;
        }
        memcpy(oldrk,rk,sizeof(rk));
        for(int i=1,p=0;i<=n;++i)
        {
            if(cmp(sa[i],sa[i-1],w))
                rk[sa[i]]=p;
            else
                rk[sa[i]]=++p;
        }
    }
} 
void get_height(int n)
{
    for(int i=1,k=0;i<=n;i++)
    {
        if(k!=0)
            k--;
        while(s[i+k]==s[sa[rk[i]-1]+k])
            k++;
        height[rk[i]]=k;
    }
}
int main()
{
    cin>>n>>k;
    m=1000000;
    for(int i=1;i<=n;i++)
        scanf("%d",&s[i]);
    SA(n,m);
    get_height(n);
    int ans=0;
    int head=0,tail=0;
    for(int i=1;i<=n;i++)
    {
        while(head<=tail&&q[head]<=i-k+1)
            head++;
        while(head<=tail&&height[q[tail]]>=height[i])
            tail--;
        q[++tail]=i;
        if(i>=k-1&&head<=tail)
            ans=max(ans,height[q[head]]);
    }
    cout<<ans<<endl;
    return 0;
}

【例题】SP220 PHRASES - Relevant Phrases of Annihilation

  $T$ 组数据，每组数据给出 $n$ 个字符串 $s_i$，找到一个最长的字符串，使得在给出的 $n$ 个字符串中，该字符串都 互不重叠 地出现了 至少两次 ，输出其长度$(T,n\leq 10,|s_i|\leq 10000)$。

分析

  将 $n$ 个字符串连起来，中间用不相同的且没有出现在字符串中的字符隔开，求后缀数组。然后二分目标串的长度 $|s|$，将 $\text{height}$ 数组划分为若干个连续 $\text{LCP}$ 大于等于 $\geq |s|$ 的段，用 $\text{ST}$ 表维护每个段中出现的数最大和最小的下标，若这两个下标的距离满足条件，则一定有长度为 $|s|$ 的字符串不重叠地出现了两次。

  设拼接后的字符串的长度为 $L=O(n+\sum\limits_{i=1}^{n}|s_i|)$，所以整个算法的时间复杂度为 $O(T\times L\log L)$。

【例题】POJ-3693 Maximum repetition substring

  给定字符串 $S(1\leq |S|\leq 10^5)$，求该字符串重复次数最多的连续重复子串（每次出现不重合）。如果有多个答案，输出字典序最小的。

分析

  先枚举长度 $L$，然后求长度为 $L$ 的子串最多能连续出现几次。连续出现 $1$ 次显然可以，所以只考虑连续出现至少 $2$ 次的情况。假设某字符串在原字符串中连续出现两次，设这个子串为 $s’$，则 $s’$ 肯定包括 $s[0],s[L],s[2L],s[3L]\cdots$ 中的某相邻的两个。所以只需要枚举相邻两字符 $s[i\ast L]$ 和 $s[(i+1)\ast L]$ 往前和往后各能匹配多远，即查询 $\text{LCP}\big(iL,(i+1)L\big)$ ，也就是区间的 $\text{height}$ 的最小值。

  往后匹配多远的 $r$ 用 $\text{ST}$ 表求 $\text{LCP}$ 即可，往前匹配多远的 $l$ 把字符串反过来再用 $\text{ST}$ 表处理即可。

  设 $\text{LCP}$ 长度为 $k$，则这个循环节至少循环了 $\lfloor\frac{k}{L}\rfloor+1 $ 次，但是当 $k\mod L\neq 0$ 时，后面多余出来的部分（长度为 $k\mod L$ 的部分）可能和前面的拼凑成循环节。所以对于位置 $i\times L-(L-k\mod L)$ 和 $(i+1)\times L-(L-k\mod L)$，求这两个位置的 $\text{LCP}$ 的长度，如果大于之前的结果，说明从这个位置开始可以比之前多一个循环节。于是可以通过枚举得到最多循环次数和循环节长度，利用后缀数组的字典序性质，找到字典序最小的即可。

结合并查集

  某些题目求解时要求你将后缀数组划分成若干个连续 $\text{LCP}$ 长度大于等于某一值的段，亦即将 数组划分成若干个连续最小值大于等于某一值的段并统计每一段的答案。如果有多次询问，可以将询问离线。观察到当给定值单调递减的时候，满足条件的区间个数总是越来越少，而新区间都是两个或多个原区间相连所得，且新区间中不包含在原区间内的部分的 $\text{height} $ 值都为减少到的这个值。只需要维护一个并查集，每次合并相邻的两个区间，并维护统计信息即可。
