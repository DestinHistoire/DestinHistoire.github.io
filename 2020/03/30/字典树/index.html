$ \text{Trie} $

  $\text{Trie}$（字典树）是一种用于实现字符串快速检索的多叉树结构。$\text{Trie}$ 的每个节点都拥有若干个字符指针，若在插入或检索字符串时扫描到一个字符 $c$，就沿着当前节点的 $c$ 字符指针，走向该指针指向的节点。下面来详细讨论 $\text{Trie}$ 的基本操作过程。

初始化

  一棵空 $\text{Trie}$ 仅包含一个根节点，该点的字符指针均指向空。

插入

  当需要插入一个字符串 $S$ 时，我们另一个指针 $P$ 起初指向根节点。然后，依次扫描 $S$ 中的每个字符 $c$：

  $1.$若 $P$ 的 $c$ 字符指针指向一个已经存在的节点 $Q$，则令 $P=Q$。

  $2.$若 $P$ 的 $c$ 字符指针指向空，则新建一个节点 $Q$，令 $P$ 的 $c$ 字符指针指向 $Q$，然后令 $P=Q$。当 $S$ 中的字符扫描完毕时，在当前节点 $P$ 上标记它是一个字符串的末尾。

检索

  当需要检索一个字符串 $S$ 在 $\text{Trie}$ 中是否存在时，我们令一个指针 $P$ 起初指向根节点，然后依次扫描 $S$ 中的每个字符 $c$：

  $1.$若 $P$ 的 $c$ 字符指针指向空，则说明 $S$ 没有被插入过 $\text{Trie}$，结束检索。

  $2.$若 $P$ 的 $c$ 字符指针指向一个已经存在的节点 $Q$，则令 $P=Q$。

  当 $S$ 中的字符扫描完毕时，若当前节点 $P$ 被标记为一个字符串的末尾，则说明 $S$ 在 $\text{Trie} $ 中存在，否则说明 $S$ 没有被插入过 $ \text{Trie} $。

  在上图所示的例子中，需要插入和检索的字符串都由小写字母构成，所以 $\text{Trie}$ 的每个节点具有 $26$ 个字符指针，分别为 $a$ 到 $z$。上图展示了在一棵空 $\text{Trie}$ 中依次插入 $“\text{cab}”,“\text{cos}”,“\text{car}”,“\text{cat}”,“\text{cate}” $ 和 “$\text{rain}$” 后的 $\text{Trie}$ 的形态，灰色标记了单词的末尾节点。可以看出在 $\text{Trie}$ 中，字符数据都体现在树的边（指针）上，树的节点仅保存一些额外信息，例如单词结尾标记等。其空间复杂度是 $O(NC)$，其中 $N$ 是节点个数，$C$ 是字符集的大小。

代码

int trie[SIZE][26],tot=1;//初始化,假设字符串由小写字母构成
void insert(char *str)//插入一个字符串 
{
    int len=strlen(str),p=1;
    for(int k=0;k<len;k++)
    {
        int ch=str[k]-'a';
        if(trie[p][ch]==0)
            trie[p][ch]=++tot;
        p=trie[p][ch];
    }
    ed[p]=true;
}
bool search(char *str)//检索字符串是否存在 
{
    int len=strlen(str),p=1;
    for(int k=0;k<len;k++)
    {
        p=trie[p][str[k]-'a'];
        if(p==0)
            return false;
    }
    return ed[p]; 
}

【例题】前缀统计

https://ac.nowcoder.com/acm/contest/1010/A

  给定 $N$ 个字符串 $S_1,S_2,\cdots,S_3,S_N $。接下来进行 $M$ 次询问，每次询问给定一个字符串 $T$ ，求 $S_1$ ~ $S_2$ 中有多少个字符串是 $T$ 的前缀。输入字符串的总长度不超过 $10^6$。

分析

  把这 $N$ 个字符串插入一棵 $\text{Trie}$ 树，$\text{Trie}$ 树的每个节点上存储一个整数 $cnt$，记录该节点是多少个字符的末尾节点。（为了处理插入重复字符串的情况，这里要记录个数，而不能只做结尾标记）

  对于每个询问，在 $\text{Trie}$ 树中检索 $T$，在检索过程中累加途径的每个节点的 $cnt$ 值，就是该询问的答案。

代码

#include<bits/stdc++.h>
using namespace std;
const int SIZE=1000010;
int trie[SIZE][26],tot=1;
int ed[SIZE];
int n,m;
char str[SIZE];
void insert(char *str)
{
    int len=strlen(str),p=1;
    for(int k=0;k<len;k++)
    {
        int ch=str[k]-'a';
        if(trie[p][ch]==0)
            trie[p][ch]=++tot;
        p=trie[p][ch];
    }
    ed[p]++;
}
int search(char *str)
{
    int len=strlen(str),p=1;
    int ans=0;
    for(int k=0;k<len;k++)
    {
        p=trie[p][str[k]-'a'];
        if(p==0)
            return ans;
        ans=ans+ed[p];
    }
    return ans;
}
int main()
{
    cin>>n>>m;
    for(int i=1;i<=n;i++)
    {
        cin>>str;
        insert(str);
    }
    for(int i=1;i<=m;i++)
    {
        cin>>str;
        cout<<search(str)<<endl;
    }
    return 0;
}

【例题】The XOR Largest Pair

https://ac.nowcoder.com/acm/contest/1010/B

  在给定的 $N$ 个整数 $A_1,A_2,\cdots,A_N$ 中选出两个进行 $\text{xor}$（异或）运算，得到的结果最大是多少？$N≤10^5,0≤A_i<2^{31}$。

分析

  考虑所有的二元组 $(i,j)$ 且 $i<j$，本题的目标就是在其中找到 $A_i\text{ xor }A_j$ 的最大值。也就是说，对于每个$ j(1\leq j\leq N) $，我们希望找到一个 $i(1\leq i<j)$，使 $A_i\text{ xor }A_j $ 最大，并求出这个最大值。

  把每个整数看作长度为 $32$ 的二进制 $01$ 串（数值较小时在前面补 $0$ ），并且把 $A_1$ ~ $A_{i-1}$ 对应的 $32$ 位二进制串插入一棵 $\text{Trie}$ 树（其中最低二进制位为叶子节点）。接下来，对于 $A_i$ 对应的 $32$ 位二进制串，我们在$ \text{Trie}$ 中进行一次与检索类似的过程，每一次都尝试沿着 “与 $A_i$ 的当前相反位相反的字符指针” 指向空节点，则只好访问与 $A_i$ 当前位相同的字符指针。根据 $\text{xor}$ 运算 “相同得 $0$，不同得 $1$” 的性质，该方法即可找出与 $A_i$ 做 $\text{xor}$ 运算结果最大的 $A_j$。

  如下图所示，在一棵插入了 $2(010),5(101),7(111)$ 三个数的 $\text{Trie}$ 中，分别查询与 $6(110),3(011)$ 做 $\text{xor}$ 运算结果最大的数（为了简便，图中使用了 $3$ 位二进制数代替 $32$ 位二进制数）。

  综上所述，我们可以循环 $1$ ~ $N$，对于每个 $i$，$\text{Trie}$ 树中存储了 $A_1$ ~ $A_{i-1}$ 对应的 $32$ 位二进制串（实际上每次 $i$ 增长前，把 $A_i$ 插入 $\text{Trie}$ 即可）。根据刚才提到的 “尽量走相反的字符指针” 的检索策略，就可以找到所求的$A_j$，更新答案。

代码

#include<bits/stdc++.h>
using namespace std;
const int N=100010;
int trie[N*30][2],tot=1;
int a[N];
void insert(int val)
{
    int p=1;
    for(int i=30;i>=0;i--)
    {
        int num=val>>i&1;
        if(trie[p][num]==0)
            trie[p][num]=++tot;
        p=trie[p][num];
    }
}
int search(int val)
{
    int p=1,ans=0;
    for(int i=30;i>=0;i--)
    {
        int num=val>>i&1;
        if(trie[p][!num]!=0)
        {
            ans=ans+(1<<i);
            p=trie[p][!num];
        }
        else
            p=trie[p][num];
    }
    return ans;
}
int main()
{
    int n;
    cin>>n;
    for(int i=1;i<=n;i++)
    {
        scanf("%d",&a[i]);
        insert(a[i]);
    }
    int ans=0;
    for(int i=1;i<=n;i++)
        ans=max(ans,search(a[i]));
    cout<<ans<<endl;
    return 0;
}

【例题】POJ-3764 The XOR Longest Path

  给定一棵 $N$ 个节点的树，树上的每条边都有一个权值。从树中选择两个点 $x$ 和 $y$，把从 $x$ 到 $y$ 的路径上的所有边权 $\text{xor}$（异或）起来，得到的结果最大是多少？$N\leq 10^5$。

分析

  设 $D[x]$ 表示根节点到 $x$ 的路径上所有边权的 $\text{xor}$ 值，显然有：
$$
D[x]=D[\text{father}(x)]\text{ xor }w\big(x,\text{father(x)}\big)
$$
   根据上式，我们可以对树进行一次深度优先遍历，求出所有的 $D[x]$。不难发现，树上 $x$ 到 $y$ 的路径上所有边权的 $\text{xor}$ 结果就等于 $D[x]\text{ xor }D[y]$。这是因为根据 $\text{xor}$ 运算的性质（$a\text{ xor }a=0 $），”$x$ 到根“ 和 ”$y$ 到根“ 这两条路径重叠的部分恰好抵消掉。

  所以，问题就变成了从 $D[1]$ ~ $D[N]$ 这 $N$ 个数中选出两个，$\text{xor}$ 的结果最大，可以用 $\text{Trie}$ 树来快速求解。

#include<bits/stdc++.h>
using namespace std;
const int N=100010;
int trie[N*30][2],tot=1;
int a[N],head[N],num_edge;
struct Edge
{
    int to;
    int dis;
    int Next;
}edge[N<<1];
void add_edge(int from,int to,int dis)
{
    edge[++num_edge].to=to;
    edge[num_edge].dis=dis;
    edge[num_edge].Next=head[from];
    head[from]=num_edge;
}
void dfs(int x,int fa,int sum)
{
    a[x]=sum;
    for(int i=head[x];i>=0;i=edge[i].Next)
    {
        int y=edge[i].to;
        if(y!=fa)
            dfs(y,x,sum^edge[i].dis); 
    }
}
void insert(int val)
{
    int p=1;
    for(int i=30;i>=0;i--)
    {
        int num=val>>i&1;
        if(trie[p][num]==0)
            trie[p][num]=++tot;
        p=trie[p][num];
    }
}
int search(int val)
{
    int p=1,ans=0;
    for(int i=30;i>=0;i--)
    {
        int num=val>>i&1;
        if(trie[p][!num])
        {
            ans=ans+(1<<i);
            p=trie[p][!num];
        }
        else
            p=trie[p][num];
    }
    return ans;
}
int main()
{
    int n;
    cin>>n;
    memset(head,-1,sizeof(head));
    for(int i=1;i<=n-1;i++)
    {
        int u,v,w;
        scanf("%d %d %d",&u,&v,&w);
        add_edge(u,v,w);
        add_edge(v,u,w);
    }
    dfs(1,-1,0);
    for(int i=1;i<=n;i++)
        insert(a[i]);
    int ans=0;
    for(int i=1;i<=n;i++)
        ans=max(ans,search(a[i]));
    cout<<ans<<endl;
    return 0;
}

可持久化 $\text{Trie}$

  若想知道数据集在任意时间的历史状态（即 $\forall i\in[1,M]$，执行完操作序列中第 $i$ 项操作后数据集的状态），一种朴素的做法是 $\forall i\in [1,M]$，在第 $i$ 项操作结束后，把整个数据结构拷贝一遍，存储在 $history[i]$ 中，多耗费 $M$ 倍的空间。”可持久化“ 提供了一种思想，在每项操作结束后，仅创建数据结构中发生改变的部分的副本，不拷贝其他部分。这样一来，维护数据结构的时间复杂度没有增加，空间复杂度仅增长为与时间同级的规模。换言之，可持久化数据结构能够高效地记录一个数据结构的所有历史状态。

  与 $\text{Trie}$ 的节点一样，可持久化 $\text{Trie}$ 的每个节点也有若干字符指针指向子节点，可以用 $trie[x,c]$ 保存节点 $x$ 的字符指针 $c$ 指向的子节点的编号（$0$ 代表指向空）。可持久化 $\text{Trie}$ 按照以下步骤插入一个新的字符串 $s$：

  $1.$ 设当前可持久化 $\text{Trie}$ 的根节点为 $root$，令 $p=root,i=0$。

  $2.$ 建立一个新的节点 $q$，令 $root’=q$。

  $3.$ 若 $p\neq 0$，则对于每种字符 $c$，令 $trie[q,c]=trie[p,c]$。

  $4.$ 建立一个新的节点 $q’$，令 $trie[q,s_i]=q’$。换言之，除了字符指针 $s_i$ 不同之外，节点 $q$ 与节点 $p$ 的其他信息完全相同。

  $5.$ 令 $p=trie[p,s_i],q=trie[q,s_i],i=i+1$。

  $6.$ 重复步骤 $3$ ~ $5$，直至 $i$ 到达字符串末尾。

  从 $root$ 出发沿着字符指针向下能够访问到的节点构成插入 $s$ 之前的 $\text{Trie}$，而从 $root’$ 出发沿着字符指针向下能够访问到的节点构成插入 $s$ 之后的 $\text{Trie}$。下图展示了在可持久化 $\text{Trie}$ 中依次插入字符串 $\text{cat,rat,cab,fry}$ 的过程。

  可以看出，可持久化 $\text{Trie}$ 的本质是一张有向图。我们把每次插入字符串后得到的新的根节点记录在数组 $root[1\text{~}4]$ 中，那么在可持久化 $\text{Trie}$ 中从 $root[i]$ 出发能够访问到的点和字符指针就组成了由前 $i$ 个字符串构成的 $\text{Trie}$。如下图所示。

  构建可持久化 $ \text{Trie} $ 所需的空间和时间复杂度都是字符串的总长度的线性函数。

【例题】P4735 最大异或和

  给定一个非负整数序列 $a$，初始长度为 $N$。有 $M$ 个操作，每个操作为以下两种类型之一：

  $1.“\text{A } x”$，添加操作，表示在序列末尾插入一个数 $x$，序列的长度 $N$ 增大 $1$。

  $2.“\text{Q}\ l\ r\ x”$，询问操作，求一个位置 $p$，满足 $l\leq p\leq r$，使得 $a[p]\text{ xor }a[p+1]\text{ xor}\cdots\text{xor }a[N]\text{ xor } x $ 最大，输出这个最大值。

  数据范围：$N,M\leq 3\times 10^5,0\leq a[i]\leq 10^7$。

分析

  设 $s[i]$ 表示序列 $a$ 的前 $i$ 个数 $\text{xor}$ 起来得到的结果。特别地，设 $s[0]=0$。根据异或运算的性质：
$$
a[p]\text{ xor }a[p+1]\text{ xor}\cdots\text{xor }a[N]\text{ xor } x=s[p-1]\text{ xor }s[N]\text{ xor }x
$$
  对于 “添加操作”，序列 $s$ 很容易维护。对于 “询问操作”，问题变为：已知一个整数 $val=s[N]\text{ xor }x$，求一个位置 $p$，满足 $l-1\leq p\leq r-1$，使得 $s[p]\text{ xor }val$ 最大。

  如果不考虑 $l-1$ 和 $r-1$ 的限制，那么就是初始有 $N+1$ 个数 $s[0]$ ~ $s[N]$，每次操作可以 “在末尾添加一个数 $s[N+1]=s[N]\text{ xor }x$ 并令 $N$ 增大 $1$” 或者 “询问序列 $s$ 中的哪个数与 $val$ 异或的结果最大”。只需把序列 $s$ 的每个数看作二进制数，以字符串的形式插入 $\text{Trie}$，询问时在 $\text{Trie}$ 中优先尝试访问与 $val$ 的每一位相反的指针即可。

  如果只有 $p\leq r-1$ 的限制，那么只需用可持久化 $\text{Trie}$ 代替上述解法中的 $\text{Trie}$。因为可持久化 $\text{Trie}$ 保存了 $\forall i\in [0,N],s[0\text{~}i]$ 这 $i+1$ 个二进制数构成的 $\text{Trie}$，所以每次询问从 $root[r-1]$ 出发，优先尝试访问与 $val$ 的每一位相反的指针即可。

  回到原题，为了要满足 $l-1\leq p\leq r-1$，我们可以在可持久化 $\text{Trie}$ 的每个节点上增加一些信息。设 $end[x]$ 表示可持久化 $\text{Trie}$ 的节点 $x$ 是序列 $s$ 中第几个二进制数的末尾节点（若不是末尾节点， $end[x]=-1$）。设 $latest[x]$ 表示在可持久化 $\text{Trie}$ 以 $x$ 为根的子树中 $end$ 的最大值。从 $root[r-1]$ 出发检索 $val$ 时，仅考虑 $latest$ 值不小于 $l-1$ 的节点即可。整个算法的时间复杂度为 $ O\Big((N+M)\log 10^7 \Big )$。

代码

#include<bits/stdc++.h>
using namespace std;
const int SIZE=600010;
int trie[SIZE*24][2],latest[SIZE*24];//latest和end可合并为一个数组 
int root[SIZE],sum[SIZE];
int n,m,tot;
//本题需要统计子树的latest，故使用递归插入s[i]，当前为s[i]的第k位 
inline void insert(int i,int k,int p,int q)
{
    if(k<0)
    {
        latest[q]=i;
        return;
    }
    int ch=(sum[i]>>k)&1;
    if(p)
        trie[q][ch^1]=trie[p][ch^1];
    trie[q][ch]=++tot;
    insert(i,k-1,trie[p][ch],trie[q][ch]);
    latest[q]=max(latest[trie[q][0]],latest[trie[q][1]]);
}
inline int ask(int now,int val,int k,int limit)
{
    if(k<0)
        return sum[latest[now]]^val;
    int ch=(val>>k)&1;
    if(latest[trie[now][ch^1]]>=limit)
        return ask(trie[now][ch^1],val,k-1,limit);
    else
        return ask(trie[now][ch],val,k-1,limit);
}
int main()
{
    cin>>n>>m;
    latest[0]=-1;
    root[0]=++tot;
    for(int i=1;i<=n;i++)
    {
        int x;
        scanf("%d",&x);
        sum[i]=sum[i-1]^x;
        root[i]=++tot;
        insert(i,23,root[i-1],root[i]);
    }
    for(int i=1;i<=m;i++)
    {
        char op[2];
        scanf("%s",op);
        if(op[0]=='A')
        {
            int x;
            scanf("%d",&x);
            root[++n]=++tot;
            sum[n]=sum[n-1]^x;
            insert(n,23,root[n-1],root[n]);
        }
        else
        {
            int l,r,x;
            scanf("%d%d%d",&l,&r,&x);
            printf("%d\n",ask(root[r-1],x^sum[n],23,l-1));
        }
    }
    return 0;
}
