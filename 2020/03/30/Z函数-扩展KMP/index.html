Z函数(扩展KMP)
字符串 扩展KMP
算法笔记
发布日期:   2020-03-30

  假设有一个长度为 $n$ 的字符串 $s$。该字符串的 Z函数 为一个长度为 $n$ 的数组，其中第 $i$ 个元素为满足从位置 $i$ 开始且为 $s$ 前缀的字符串的最大长度，即 $z[i]$ 是 $s$ 和从 $i$ 开始的 $s$ 的后缀的最长公共前缀的长度。$\text{Z}$ 函数的第一个元素 $z[0]$ 通常是无意义的，假定 $z[0]=0$。

  $\text{Z}$ 函数举例如下：

朴素算法

  按照 $\text{Z}$ 函数的定义，可写出下列代码，时间复杂度为 $O(n^2)$。

void Z_algorithm()
{
    for(int i=1;i<n;i++)
    {
        while(i+z[i]<n&&s[z[i]]==s[i+z[i]])
            z[i]++;
    }
}

  我们所做的仅仅是循环每个位置 $i$，并通过如下做法更新每个 $z[i]$：从 $z[i]=0$ 开始，只要没有失配（并且没有到达末尾），就将 $z[i]$ 加 $1$。显然，这并不是一个高效的实现。

计算 $\text{Z}$ 函数的高效算法

  为了得到一个高效算法，我们将以 $i=1$ 到 $i=n-1$ 的顺序计算 $z[i]$，在计算一个新值的同时，我们将尝试尽最大努力使用之前已经计算好的值。

  为了简便起见，定义 匹配段 为和 $s$ 一个前缀相同的那些子串。举例来说，所求 $\text{Z}$ 函数的第 $i$ 个元素 $z[i]$ 为从位置 $i$ 开始的匹配段的长度（其终止位置位于 $i+z[i]-1$）。

  为了达成目标，我们将始终保持 $[l,r]$ 为最靠右的匹配段。也就是说，在所有已探测到的匹配段中，我们将保持结尾最靠右的那一个。另一方面，下标 $r$ 可被认为是字符串 $s$ 已被算法扫描的边界；任何超过该点的字符都是未知的。

  假设当前下标为 $i$（即我们要计算的下一个 $\text{Z}$ 函数值的下标），则有两种情况：

  $1.i>r$——当前位置在我们已处理位置之外。

  接下来使用 朴素算法（即一个一个的比较字符）来计算 $z[i]$。注意如果最后 $z[i]>0$，需要更新最靠右的匹配段的下标，因为新的 $r=i+z[i]-1$ 一定比之前的 $r$ 优。

  $2.i\leq r$——当前位置位于当前匹配段 $[l,r]$ 之内。

  那么我们可以用已经计算过的 $\text{Z}$ 函数值来 “初始化” $z[i]$ 至某值（至少比从 $0$ 开始要好），甚至可能是某些较大的值。

  为了做到这一点，可以注意到子串 $s[l\cdots r]$ 和 $s[0\cdots r-l]$ 匹配。这意味着作为 $z[i] $ 的一个初始近似，可以直接使用对应于子串 $s[0\cdots r-l]$ 的已经计算过的 $\text{Z}$ 函数的值，也即 $z[i-l]$。

  然而 $z[i-l]$ 可能非常大：将其应用到位置 $i$ 结果可能超过下标 $r$。这种做法并不合法，原因在于我们对 $r$ 右侧的字符一无所知：它们可能并不满足要求。

  比如：
$$
s=\text{aaaabaa}
$$
  当我们尝试计算末尾位置（$i=6$）的值时，当前匹配的段为 $[5,6]$。位置 $6$ 会匹配位置 $6-5=1$，其 $\text{Z}$ 函数值为 $z[1]=3$。显然不能将 $z[6]$ 初始化为 $3$。但是我们可以初始化最大值为 $1$——这是使我们不超过段 $[l,r]$ 的边界 $r$ 的最大可能取值。

  因此，可以将下列值作为 $z[i]$ 的一个初始近似：
$$
z_0[i]=\min\big(r-i+1,z[i-l]\big)
$$

  当将 $z[i]$ 初始化为 $z_0[i]$ 后，我们尝试使用 朴素算法 增加 $z[i]$ 的值——因为宏观来讲，对于边界 $r$ 之后的字符，无法得知段是否会继续匹配还是失配。

  综上所述，整个算法被划分成两种情况，他们只在设置 $z[i]$ 的 初始值 时有所不同：在第一种情况下，其被认为为 $0$，在第二种情况下它由先前已计算过的值确定（使用前述公式）。之后，该算法的两个分支都被规约为实现 朴素算法 。当我们设置完初始值后，该算法即开始执行。

  尽管在每轮迭代后都会运行朴素算法，但是其时间复杂度为 $O(n)$。

代码

  当前最右的匹配段假定为 $[0,0]$（不包含任何 $i$ 的小段）。在循环 $i=1\cdots n-1$ 中，首先确定 $z[i]$ 的初始值——要么保持为 $0$，要么使用前述公式进行计算。之后，朴素算法尝试尽可能多的增加 $z[i]$ 值。最后，如果 $i+z[i]-1>r$，更新最右匹配段 $[l,r]$。

void Z_algorithm(char *s,int n)
{
    z[1]=n;
    int l=0,r=0;
    for(int i=2;i<=n;i++)
    {
        if(i<=r)
            z[i]=min(z[i-l+1],r-i+1);
        while(i+z[i]<=n&&s[i+z[i]]==s[z[i]+1])
            z[i]++;
        if(i+z[i]-1>r)
        {
            l=i;
            r=i+z[i]-1;
        }
    }
}

算法的时间复杂度证明

  只考虑内层的 $\text{while}$ 循环，因为其余部分在一次循环中只是一堆常数次操作，时间复杂度总和为 $O(n)$。

  下面将证明 $\text{while}$ 的每次迭代都将增加匹配段的右边界 $r$。

  考虑算法的所有分支：

  $1.i>r$

  在这种情况下，要么 $\text{while}$ 循环不进行任何迭代（如果 $s[0]\neq s[i]$），要么其将从位置 $i$ 开始进行若干次迭代，其中每次迭代将向右移动一个字符。每次迭代后，右边界 $r$ 必定被更新。

  因此当 $i>r$ 时，$\text{while}$ 循环的每轮迭代都会使新的 $r$ 增加 $1$。

  $ 2.i\leq r $

  在这种情况下，将 $z[i]$ 初始化为前述公式给出的某个具体 $z_0$。将 $z_0$ 和 $r-i+1$ 比较，可能有三种情况：

  ①$z_0<r-i+1$

  在这种情况下 $\text{while}$ 循环不会进行任何迭代。

  运用反证法：如果 $\text{while}$ 循环进行了至少一次迭代，这意味着初始近似 $z[i]=z_0$ 是不准确的（小于匹配的实际长度）。但是由于 $s[l\cdots r]$ 和 $s[0\cdots r-l]$ 是相同的，因此 $z[i-l]$ 的值是错误的（比其应有的值要小）。

  因为 $z[i-l]$ 的值是正确的且 $z[i-l]<r-i+1$，所以此值与所求的 $z[i]$ 是相同的。

  ②$z_0=r-i+1$

  在这种情况下，$\text{while}$ 循环可能会进行若干次迭代。因为我们从 $s[r+1]$ 开始比较，其位置已经超过了区间 $[l,r]$，故每次迭代都会使 $r$ 增加。

  ③$z_0>r-i+1$

  根据 $z_0$ 的定义，这种情况不可能。

  综上，我们已经证明了内层循环的每次迭代都会使 $r$ 向右移动，由于 $r$ 不可能超过 $n-1$，因此内层 $\text{while}$ 循环至多进行 $n-1$ 轮迭代。

  所以算法的时间复杂度为 $O(n)$。

P5410【模板】扩展 KMP（Z 函数）

  给定两个字符串 $a,b$，求出两个数组：

  $b$ 的 $\text{Z}$ 函数数组 $z$，即 $b$ 与 $b$ 的每一个后缀的 $ \text{LCP} $ 长度。

  $b$ 与 $a$ 的每一个后缀的$\text{ LCP }$长度数组 $p$。

  对于一个长度为 $n$ 的数组 $a$，设其权值为 $\text{xor}_{i=1}^n i \times (a_i+1)$。

  输出 $z$ 的权值与 $p$ 的权值$(1\leq |a|,|b|\leq 2\times 10^7)$。

代码

#include<iostream>
#include<cstdio>
#include<cstring>
using namespace std;
const int N=2e7+10;
int n,m,z[N],p[N];
char a[N],b[N];
long long ans;
void Z_algorithm(char *s,int n)
{
    z[1]=n;
    int l=0,r=0;
    for(int i=2;i<=n;i++)
    {
        if(i<=r)
            z[i]=min(z[i-l+1],r-i+1);
        while(i+z[i]<=n&&s[i+z[i]]==s[z[i]+1])
            z[i]++;
        if(i+z[i]-1>r)
        {
            l=i;
            r=i+z[i]-1;
        }
    }
}
void exKMP(char *s,char *t,int n)
{
    int l=0,r=0;
    for(int i=1;i<=n;i++)
    {
        if(i<=r)
            p[i]=min(z[i-l+1],r-i+1);
        while(i+p[i]<=n&&s[i+p[i]]==t[p[i]+1])
            p[i]++;
        if(i+p[i]-1>r)
        {
            l=i;
            r=i+p[i]-1;
        }
    }
}
int main()
{
    scanf("%s%s",a+1,b+1);
    n=strlen(a+1);m=strlen(b+1);
    Z_algorithm(b,m);
    ans=0;
    for(int i=1;i<=m;i++)
        ans^=1ll*i*(z[i]+1);
    printf("%lld\n",ans);
    ans=0;
    exKMP(a,b,n);
    for(int i=1;i<=n;i++)
        ans^=1ll*i*(p[i]+1);
    printf("%lld\n",ans);
    return 0;
}
