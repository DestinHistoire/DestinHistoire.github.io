前缀函数与KMP算法
字符串 KMP算法
算法笔记
发布日期:   2020-03-27

前缀函数定义

  给定一个长度为 $n$ 的字符串 $s$，其 前缀函数 被定义为一个长度为 $n$ 的数组 $\text{next}$。其中 $\text{next}[i]$ 表示既是子串 $s[0\cdots i]$ 的前缀，同时也是子串 $s[0\cdots i]$ 的后缀的最长真前缀（$\text{proper prefix}$）长度。一个字符串的真前缀是该字符串的前缀但不等于该字符串自身。根据定义，$\text{next}[0]=0$。

  前缀函数的定义可用数学语言描述如下：

  当不存在这样的 $j$ 时，令 $\text{next}[i]=0$。

  举例来说，字符串 $\text{abcabcd}$ 的前缀函数为 $[0,0,0,1,2,3,0]$，字符串 $\text{aabaaab}$ 的前缀函数为 $[0,1,0,1,2,2,3]$。

朴素算法

  一个直接按照定义计算前缀函数的算法如下：

void get_next(string s)
{
    int n=s.length();
    for(int i=0;i<n;i++)
        for(int j=0;j<=i;j++)
            if(s.substr(0,j)==s.substr(i-j+1,j))
                Next[i]=j; 
}

  时间复杂度为 $O(n^3)$。

高效算法

第一个优化

  可以观察到：相邻的前缀函数值至多增加 $1$。

  证明：假设该结论不成立，则 $\text{next}[i]+1<\text{next}[i+1]$。考虑长度为 $\text{next}[i+1]$ 的 $s[0\cdots i+1]$ 的后缀可发现矛盾。该后缀去掉最后一个字符后，得到一个长度为 $\text{next}[i+1]-1 $ 的 $s[0\cdots i]$ 的后缀。$s[0\cdots i]$ 这个后缀的长度大于 $\text{next}[i]$，与前缀函数的定义产生矛盾，因此假设不成立。

  下图展示了这个矛盾。假设位于位置 $i$ 和 $i+1$ 的既是后缀又是前缀的最长真后缀的长度分别为 $2$ 和 $4$。则字符串 $s_0s_1s_2s_3$ 与字符串 $s_{i-2}s_{i-1}s_{i}s_{i+1}$ 相同，这意味着 $s_0s_1s_2$ 与字符串 $s_{i-2}s_{i-1}s_i$ 相同，因此 $\text{next}[i]$ 至少为 $3$。

  所以当移动到下一个位置时，前缀函数的值要么增加 $1$，要么维持不变，要么减少。实际上，该事实已经允许我们将算法的时间复杂度减少至 $O(n^2)$。因为每步中前缀函数至多增加 $1$，因此在总的运行过程中，前缀函数至多增加 $n$，同时也至多减小 $n$。这意味着我们仅需要进行 $O(n)$ 次字符串比较，所以总时间复杂度为 $O(n^2)$。

第二个优化

  尝试摆脱掉字符串比较。为了达成这一点，我们必须用到先前计算的所有信息。

  考虑计算位置 $i+1$ 的前缀函数 $\text{next}[i+1]$ 的值。如果 $s[i+1]=s[\text{next}[i]]$，那么我们可以断言 $\text{next}[i+1]=\text{next[i]}+1$，因为我们已经知道位于位置 $i$ 的长度为 $\text{next}[i]$ 的后缀同长度为 $\text{next}[i]$ 的前缀相等。参考下图：

  如果不是上述情况，即 $s[i+1]\neq s[\text{next}[i]]$，那么我们需要尝试更短的字符串。为了加速，我们希望直接移动到最长的长度 $j<\text{next}[i]$，使得在位置 $i$ 的前缀性质仍得以保持，也即 $s[0\cdots j-1]=s[i-j+1\cdots i]$。

  如果我们找到了这样的长度 $j$，仅需要再次比较 $s[i+1]$ 和 $s[j]$。如果 $s[i+1]=s[j]$，那么令 $\text{next}[i+1]=j+1$。否则，我们需要找到小于 $j$ 的最大值使得前缀性质得以保持，如此反复。这个过程会一直持续，直到 $ j=0 $，如果 $s[i+1]=s[0]$，那么我们令 $\text{next}[i+1]=1$，否则 $\text{next}[i+1]=0$。

  现在仅剩的问题是对于 $j$，如何快速找到这样的长度。也就是说：对于当前在位置 $i$ 使得前缀性质得以保持的长度 $j$，即 $s[0\cdots j-1]=s[i-j+1\cdots i]$，我们希望找到最大的 $k<j$，使得前缀性质仍得以保持。

  上图显示出，$k$ 必定为 $\text{next}[j-1]$，而该值我们之前已经计算过了。

最终算法

  最终可以构建一个不需要进行任何字符串比较，并且只进行 $O(n)$ 次操作的算法。

  以下是最终的算法流程：

  $1.$在一个循环中以 $i=1$ 到 $i=n-1$ 的顺序计算前缀函数 $\text{next}[i]$ 的值（$\text{next}[0]$ 被赋值为 $0$）。

  $2.$为了计算当前的前缀函数值 $\text{next}[i]$，我们令变量 $j$ 表示右端点位于 $i-1$ 的最好的后缀的长度。初始时，$j=\text{next}[i-1]$。

  $3.$通过比较 $s[j]$ 和 $s[i]$ 来检查长度为 $j+1$ 的后缀是否同时也是一个前缀。如果二者相等，那么令 $\text{next}[i]=j+1$，否则我们减少 $j$ 至 $\text{next}[j-1]$ 并且重复该过程。

代码

void get_next()
{
    int j=0;
    for(int i=1;i<n;i++)
    {
        while(j>0&&s[i]!=s[j])
            j=Next[j-1];
        if(s[i]==s[j])
            j++;
        Next[i]=j;
    }
}

  这是一个 在线 算法，也就是说一个字符一个字符的读取字符串后，可以立即处理它们以计算出每个字符的前缀函数值。该算法仍然需要存储字符串本身以及先前计算过的前缀函数值，但如果我们已经预先知道该字符串前缀函数的最大可能取值 $M$，那么我们仅需要存储该字符串的前 $M+1$ 个字符以及对应的前缀函数值。

在字符串中查找子串：Knuth-Morris-Pratt 算法

  给定一个字符串 $s$ 和一个字符串 $t$，判断 $t$ 是否为 $s$ 的子串，并求出 $t$ 在 $s$ 中各次出现的位置。

  $\text{KMP}$ 算法分为两步：

  $1.$对字符串 $t$ 进行自我 “匹配”，求出前缀函数 $\text{next}$，其中 $\text{next}[i]$ 表示既是子串 $s[0\cdots i]$ 的前缀，同时也是子串 $s[0\cdots i]$ 的后缀的最长真前缀长度，即：

  $2.$ 对字符串 $s$ 与 $t$ 进行匹配，求出一个数组 $f$，其中 $f[i]$ 表示 ”$s$ 中以 $i$ 结尾的子串“ 与 ”$t$ 的前缀“ 能够匹配的最长长度，即：

  因为定义的相似性，求解 $f$ 数组与求解 $\text{next}$ 数组的过程是基本一致的。

void KMP()
{
    int j=0;
    for(int i=0;i<n;i++)
    {
        while(j>0&&s[i]!=t[j])
            j=Next[j-1];
        if(s[i]==t[j])
            j++;
        f[i]=j;
        if(f[i]==m)//此时就是t在s中的某一次出现
            printf("%d\n",i-m+1+1);//如果串的下标位置从1开始需要+1
    }
}

  这就是 $\text{KMP}$ 模式匹配算法。在上面代码的 $\text{while}$ 循环中，$j$ 的值不断减小，$j=\text{Next}[j-1]$ 的执行次数不会超过每层 $\text{for}$ 循环开始时 $j$ 的值与 $\text{while}$ 循环结束时 $j$ 的值之差。而在每层 $\text{for}$ 循环中，$j$ 的值至多增加 $1$。因为 $j$ 始终非负，所以在整个计算过程中，$j$ 减小的幅度总和不会超过 $j$ 增加的幅度总和。故 $j$ 的总变化次数至多为 $2(N+M)$。整个算法的时间复杂度为 $O(N+M)$。

P3375 【模板】KMP字符串匹配

  给出两个字符串 $s_1$ 和 $s_2$，其中 $s_2$ 为 $s_1$ 的子串，求出 $s_2$ 在 $s_1$ 中所有出现的位置（$|s_1|,|s_2|\leq 10^6$）。

#include<bits/stdc++.h>
using namespace std;
char s[1000010],t[1000010];
int n,m,cnt;
int Next[1000010],f[1000010];
void get_next()
{
    int j=0;
    for(int i=1;i<m;i++)
    {
        while(j>0&&t[i]!=t[j])
            j=Next[j-1];
        if(t[i]==t[j])
            j++;
        Next[i]=j;
    }
}
void KMP()
{
    int j=0;
    for(int i=0;i<n;i++)
    {
        while(j>0&&s[i]!=t[j])
            j=Next[j-1];
        if(s[i]==t[j])
            j++;
        if(j==m)
            f[++cnt]=i-m+1+1;
    }
}
int main()
{
    scanf("%s",s);
    scanf("%s",t);
    n=strlen(s);
    m=strlen(t);
    get_next();
    KMP();
    for(int i=1;i<=cnt;i++)
        cout<<f[i]<<endl;
    for(int i=0;i<m;i++)
        cout<<Next[i]<<" ";
    cout<<endl;
    return 0;
}

KMP算法求字符串最小循环节长度

【例题】POJ-1961 Period

  如果一个字符串 $S$ 是由一个字符串 $T$ 重复 $K$ 次形成的，则称 $T$ 是 $S$ 的循环节。使 $K$ 最大的字符串 $T$ 称为 $S$ 的最小循环节，此时的 $K$ 称为最大循环次数。

  现在给定一个长度为 $N$ 的字符串 $S$，对 $S$ 的每个前缀 $S[1\cdots i]$，如果它的最大循环次数大于 $1$，则输出该前缀的最小循环元长度和最大循环次数$(2\leq |S|\leq 10^6)$。

分析

  对字符串 $S$ 自匹配求出 $\text{next}$ 数组，根据定义，对于每个 $i$，$S[i-\text{next}[i]+1\cdots i]$ 与 $S[1\cdots \text{next[i]}]$ 是相等的，并且不存在更大的 $ \text{next} $ 值满足这个条件。

引理

  $S[1\cdots i]$ 具有长度为 $len<i$ 的循环节的充要条件是 $len$ 能整除 $i$ 并且 $S[len+1\cdots i]=S[1\cdots i-len]$。

证明：

  先证必要性。设 $S[1\cdots i]$ 具有长度为 $len$ 的循环节，显然 $len$ 能整除 $i$，并且 $S[1\cdots i-len]$ 和 $S[len+1\cdots i]$ 都是由 $i/len-1$ 个循环节构成的。故 $S[1\cdots 1-len]=S[len+1\cdots i]$。

  再证充分性。设 $len$ 能整除 $i$ 并且 $S[len+1\cdots i]=S[1\cdots i-len]$。 因为 $len<i$，所以 $S[1\cdots i-len]$ 和 $S[len+1\cdots i ] $ 的长度不小于 $len$ 且是 $len$ 的倍数。二者各取前 $len$ 个字符，有 $S[1\cdots len]=S[len+1\cdots 2\ast len]$。依次类推，不断向后取 $len$ 个字符，可以发现 $ S[1\cdots i-len] $ 和 $S[len+1\cdots i] $ 是以 $len$ 为间隔错位对齐的，如下图所示。故 $S[1\cdots len]$ 是 $S$ 的循环节。

  根据引理，当 $i-\text{next}[i]$ 能整除 $i$ 时，$S[1\cdots i-\text{next}[i]]$ 就是 $S[1\cdots i]$ 的最小循环节。它的最大循环次数就是 $i/(i- \text{next}[i])$。其中 $i-\text{next}[i]$ 能整除 $i$ 的条件是为了保证循环节每次重复的完整性。例如上图中 $i=7$ 时，最小循环节 $\text{“ab”}$ 的第 $4$ 次重复就尚未完成。

  进一步地，如果 $i-\text{next}\big[\text{next}[i] \big] $ 能整除 $i$，那么 $S[1\cdots i-\text{next[next[i]]]}$ 就是 $S[1\cdots i]$ 的次小循环元。依次类推，我们还可以找出 $S[1\cdots i]$ 所有可能的循环节。

  值得指出的一个性质是：一个字符串的任意循环节的长度必然是最小循环节长度的倍数。

代码

字符串下标从 $1$ 开始

#include<cstdio>
#include<cstring>
#include<iostream>
using namespace std;
char str[1000010];
int Next[1000010],n;
void get_next()
{
    int j=0;
    for(int i=2;i<=n;i++)
    {
        while(j>0&&str[i]!=str[j+1])
            j=Next[j];
        if(str[i]==str[j+1])
            j++;
        Next[i]=j;
    }
}
int main()
{
    int kase=0;
    while(cin>>n&&n)
    {
        scanf("%s",str+1);
        get_next();
        printf("Test case #%d\n",++kase);
        for(int i=2;i<=n;i++)
        {
            if(i%(i-Next[i])==0&&i/(i-Next[i])>1)
                printf("%d %d\n",i,i/(i-Next[i]));
        }
        puts("");
    }
    return 0;
}

KMP算法统计每个前缀的出现次数

【例题】HDU-3336 Count the String

  给定一个字符串 $S(1\leq |S|\leq 2\times 10^5)$，求其所有前缀在字符串 $S$ 中出现次数的总和，答案对 $10007$ 取模。

分析

  设 $\text{dp}[i]$ 表示以 $i$ 结尾的字符串中前缀相同的串的个数。

  考虑位置 $i$（假设字符串起始位置为 $0$） 的前缀函数值 $\text{next}[i]$。根据定义，这意味着字符串 $s$ 一个长度为 $\text{next}[i] $ 的前缀在位置 $i$ 出现并以 $i$ 为右端点，同时不存在一个更长的前缀满足前述定义。与此同时，更短的前缀可能以该位置为右端点。此时需要寻找下一个更小的长度为 $k$ 且 $k<j$ 的前缀，该长度的前缀同时也是一个右端点为 $i$ 的后缀。因此以位置 $i$ 为右端点，有长度为 $\text{next}[i]$ 的前缀，有长度为 $\text{next[next}[i]-1]$ 的前缀，有长度为 $\text{next[next[next}[i] -1]-1]$ 的前缀，等等，直到长度变为 $0$。所以可以通过下述方式计算答案（为了统计原始的前缀，需要对每个结果加 $1$）：

int ans=0;
for(int i=1;i<=n;i++)
{
    dp[i]=dp[Next[i-1]]+1;
    ans=ans+dp[i];
}

代码

#include<bits/stdc++.h>
using namespace std;
const int mod=10007;
char s[1000010];
int Next[1000010],dp[1000010],n;
void get_next()
{
    int j=0;
    for(int i=1;i<n;i++)
    {
        while(j>0&&s[i]!=s[j])
            j=Next[j-1];
        if(s[i]==s[j])
            j++;
        Next[i]=j;
    }
}
int main()
{
    int T;
    cin>>T;
    while(T--)
    {
        scanf("%d%s",&n,s);
        get_next();
        int ans=0;
        for(int i=1;i<=n;i++)
        {
            dp[i]=(dp[Next[i-1]]+1)%mod;//S[0...i]本身也是一个前缀,所以要+1
            ans=(ans+dp[i])%mod;
        } 
        printf("%d\n",ans);
    }
    return 0;
}
