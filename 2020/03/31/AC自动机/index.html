算法笔记
发布日期:   2020-03-31

自动机

  自动机是一种理想化的 “机器”，它只是抽象分析问题的理论工具，并不具有实际的物质形态。它是科学定义的演算机器，用来表达某种不需要人力干涉的机械性演算过程。在算法竞赛中，我们所说的 “自动机” 一般都指 “确定有限状态自动机”。

  一个 确定有限状态自动机（DFA）由以下五部分构成：

  $1.$ 字符集（$\sum$）。该自动机只能输入这些字符。

  $2.$ 状态集合 （$Q$）。如果把一个 $\text{DFA}$ 看成一张有向图，那么 $\text{DFA}$ 中的状态就相当于图上的顶点。

  $3.$ 起始状态（$start$），$start\in Q$，是一个特殊的状态。起始状态一般用 $s$ 表示，为了避免混淆，本文中使用 $start$。

  $4.$ 接受状态集合（$F$），$F\subseteq Q$，是一组特殊的状态。

  $5.$ 转移函数（$\delta$），$\delta$ 是一个接受两个参数，返回一个值的函数，其中第一个参数和返回值都是一个状态，第二个参数是字符集中的一个字符。如果把一个 $\text{DFA}$ 看成一张有向图，那么 $\text{DFA}$ 中的转移函数就相当于顶点间的边，而每条边上都有一个字符。

  $\text{DFA}$ 的作用就是识别字符串，一个自动机 $A$，若它能识别（接受）字符串 $S$，那么 $A(S)=\text{true}$，否则 $A(S)=\text{false}$。

  当一个 $\text{DFA}$ 读入一个字符串时，从初始状态起按照转移函数一个一个字符的转移。如果读入完一个字符串的所有字符后位于一个接受状态，那么我们称这个 $\text{DFA}$ 接受 这个字符串，反之我们称这个 $\text{DFA}$ 不接受 这个字符串。

  如果一个状态 $v$ 没有字符 $c$ 的转移，那么我们令 $\delta(v,c)=\text{null}$，而 $\text{null}$ 只能转移到 $\text{null}$，且 $\text{null}$ 不属于接受状态集合。无法转移到任何一个接受状态的状态都可以视作 $\text{null}$，或者说，$\text{null}$ 代指所有无法转移到任何一个接受状态的状态。

  扩展定义转移函数 $\delta$，令其第二个参数可以接收一个字符串：

  $\delta(v,s)=\delta(\delta(v,s[1]),s[s\cdots |s|])$，扩展后的转移函数就可以表示从一个状态起接收一个字符串后转移到的状态。那么，$A(s)=[\delta(start,s)\in F]$。

AC自动机

概述

  $\text{AC}$自动机是以 $\text{Trie}$ 的结构为基础，结合 $\text{KMP}$ 的思想建立的。

  简单来说，建立一个 $\text{AC}$ 自动机有两个步骤：

  $1.$基础的 $ \text{Trie} $ 结构：将所有的模式串构成一棵 $ \text{Trie }$。

  $2.\text{KMP}$ 的思想：对 $ \text{Trie} $ 树上所有的结点构造失配指针。

  然后就可以利用它进行多模式匹配了。

字典树构建

  $\text{AC}$自动机在初始时会将若干个模式串丢到一个 $\text{Trie}$ 里，然后在 $\text{Trie}$ 上建立 $\text{AC}$ 自动机。在这里，$\text{Trie}$ 中的节点表示的是某个模式串的前缀，我们在后文也将其称作状态。一个节点表示一个状态，$\text{Trie}$ 中的边就是状态的转移。

  形式化地说，对于若干个模式串 $s_1,s_2,\cdots,s_n$，将它们构建一棵字典树后的所有状态的集合记作 $Q$。

失配指针

  $\text{AC}$自动机利用一个 $\text{fail}$ 指针来辅助多模式串的匹配。

  状态 $u$ 的 $\text{fail}$ 指针指向另一个状态 $v$，其中 $v\in Q$，且 $v$ 是 $u$ 的最长后缀（即在若干个后缀状态中取最长的一个作为 $\text{fail}$ 指针）。

  对比这里的 $\text{fail}$ 指针与 $\text{KMP}$ 中的 $\text{next}$ 指针：

  $1.$ 共同点：两者同样是在失配的时候用于跳转的指针。

  $2.$ 不同点：$\text{next}$ 指针求的是最长相同前后缀，而 $\text{fail}$ 指针指向所有模式串的前缀中匹配当前状态的最长后缀。

  因为 $\text{KMP}$ 只对一个模式串做匹配，而 $\text{AC}$ 自动机要对多个模式串做匹配。有可能 $\text{fail}$ 指针指向的结点对应着另一个模式串，二者前缀不同。

  $\text{AC}$自动机在做匹配时，同一位上可匹配多个模式串。

构建指针

  构建 $\text{fail}$ 指针，可以参考 $\text{KMP}$ 中构造 $\text{next}$ 指针的思想。

  考虑字典树中当前的结点 $u$，$u$ 的父节点是 $p$，$p$ 通过字符 $c$ 的边指向 $u$，即 $\text{trie}[p][c]=u$。假设深度小于 $u$ 的所有节点的 $\text{fail}$ 指针都已求得。

  $1.$ 如果 $ \text{trie}[\text{fail}[p]][c] $ 存在：则让 $u$ 的指针指向 $\text{trie}[\text{fail}[p]][c]$。相当于在 $p$ 和 $\text{fail}[p]$ 后面加一个字符 $c$，分别对应 $u$ 和 $\text{fail}[u]$。

  $2.$ 如果 $\text{trie}[\text{fail}[p]][c] $ 不存在：那么我们继续找到 $\text{trie}[\text{fail}[\text{fail}[p]]][c]$，重复 $1$ 的判断过程，一直跳 $\text{fail}$ 指针直到根节点。

  $3.$ 如果真的没有，就让 $\text{fail} $ 指针指向根节点。

  如此即完成了 $\text{fail}[u]$ 的构建。

举例

  对字符串 $\text{i he his she hers}$ 组成的字典树构建 $\text{fail}$ 指针：

  $1.$ 黄色结点：当前的节点 $u$。

  $2.$ 绿色节点：表示已经 $\text{BFS}$ 遍历完毕的结点。

  $3.$ 橙色的边：$\text{fail}$ 指针。

  $4.$ 红色的边：当前求出的 $\text{fail}$ 指针。

  重点分析结点 $6$ 的 $\text{fail}$ 指针构建：

  找到 $6$ 的父节点 $5$，$ \text{fail}[5]=10 $。然而 $10$ 节点没有字母 $s$ 连出的边；继续跳到 $10$ 的 $\text{fail}$ 指针，$\text{fail}[10]=0$。发现 $0$ 节点有字母 $s$ 连出的边，指向 $7$ 节点；所以 $\text{fail}[6]=7$。最后放一张建出来的图片：

字典树与字典图

  先看构建函数 $\text{build}()$，该函数的目标有两个，一个是构建 $\text{fail}$ 指针，一个是构建自动机。参数如下：

  $1.$ $\text{trie}[u][c]$ 有两种理解方式。可以简单理解为字典树上的一条边，即 $\text{trie}[u][c]$；也可以理解为从状态（节点）$u$ 后加一个字符 $c$ 到达的状态（节点），即一个状态转移函数 $trans(u,c) $。

  下文中将用第二种理解方式继续讲解：

  $2.\text{Q}$ 队列用于 $\text{BFS}$ 遍历字典树。

  $3.$ $\text{fail}[u]$ 表示节点 $u$ 的 $\text{fail}$ 指针。

void build()
{
    for(int i=0;i<26;i++)
        if(trie[0][i]!=0)
            Q.push(trie[0][i]);
    while(!Q.empty())
    {
        int u=Q.front();
        Q.pop();
        for(int i=0;i<26;i++)
        {
            if(trie[u][i]!=0)
            {
                fail[trie[u][i]]=trie[fail[u]][i];
                Q.push(trie[u][i]);
            }
            else
                trie[u][i]=trie[fail[u]][i];
        }
    }
}

  $\text{build}$ 函数将节点按 $\text{BFS}$ 顺序入队，依次求 $\text{fail}$ 指针。这里的字典树根节点为 $0$，我们将根节点的子节点一一入队。若将根节点入队，则在第一次 $\text{BFS}$ 的时候，会将根节点儿子的 $\text{fail}$ 指针标记为本身。因此我们将根节点的儿子一一入队，而不是将根节点入队。

   然后开始 $\text{BFS}$：每次取出队首的节点 $u$。$ \text{fail}[u] $ 指针已经求得，我们要求 $u$ 的所有子节点的 $\text{fail} $ 指针，然后遍历字符集（$0$ ~ $25$，对应 $a$ ~ $z$）：

  $1.$ 如果 $trans(u,i)$ 存在，我们就将 $trans(u,i)$ 的 $\text{fail}$ 指针赋值为 $trans(\text{fail}[u],i)$。根据之前的讲解，似乎应该用 $\text{while}$ 循环，不停地跳 $\text{fail}$ 指针，判断是否存在字符 $i$ 对应的节点，然后赋值。不过在代码中一行就完成了这个操作。

  $2.$ 否则，$trans(u,i)$ 不存在，就让 $trans(u,i)$ 指向 $trans(fail[u],i)$ 的状态。

  可以发现，上面代码中的 $\text{else}$ 语句会修改字典树的结构，它将不存在的字典树的状态链接到了失配指针的对应状态。在原字典树中，每一个节点代表一个字符串 $S$，是某个模式串的前缀。而在修改字典树结构后，尽管增加了许多转移关系，但节点（状态）所代表的字符串是不变的。

  而 $trans(S,c)$ 相当于是在 $S$ 后添加一个字符 $c$ 变成另一个状态 $S’$。如果 $S’$ 存在，说明存在一个模式串的前缀是 $S’$，否则我们让 $trans(S,c)$ 指向 $trans(\text{fail}[S],c)$。由于 $\text{fail}[S]$ 对应的字符串是 $S$ 的后缀，因此 $trans(\text{fail}[S],c)$ 对应的字符串也是 $S’$ 的后缀。

  换言之，在 $\text {Trie}$ 上跳转的时侯，我们只会从 $S$ 跳转到 $S’$，相当于匹配了一个 $ S’ $；但在$\text{AC}$ 自动机上跳转的时侯，我们会从 $S$ 跳转到 $ S’ $ 的后缀，也就是说我们匹配一个字符 $c$，然后舍弃 $S$ 的部分前缀。舍弃前缀显然是能匹配的。而 $\text{fail}$ 指针也是在舍弃前缀，如果文本串能匹配 $S$，显然它也能匹配 $S$ 的后缀。所谓的 $\text{fail}$ 指针其实就是 $S$ 的一个后缀集合。

  $\text{trie}$ 数组还有另一种比较简单的理解方式：如果在 $u$ 的位置失配，我们会跳转到 $\text{fail}[u] $ 的位置。所以我们可能沿着 $\text{fail}$ 数组跳转多次才能来到下一个能匹配的位置。所以我们可以用 $\text{trie}$ 数组直接记录记录下一个能匹配的位置，这样就能节省下很多时间。

  这样修改字典树的结构，使得匹配转移更加完善。同时它将 $\text{fail}$ 指针跳转的路径做了压缩（就像并查集的路径压缩），使得本来需要跳很多次 $\text{fail}$ 指针变成跳一次。

  $1. $蓝色节点：$\text{BFS}$ 遍历到的节点 $u$。

  $2.$蓝色的边：当前节点下，$\text{AC}$自动机修改字典树结构连出的边。

  $3.$黑色的边：$\text{AC}$自动机修改字典树结构连出的边。

  $4.$红色的边：当前节点求出的 $\text{fail}$ 指针。

  $5.$黄色的边：字典树的边。

  可以发现，众多交错的黑色边将字典树变成了 字典图。图中省略了连向根节点的黑色边。

  重点分析节点 $5$ 遍历时的情况，求 $trans(5,s)=6$ 的 $\text{fail}$ 指针。

  本来的策略是找 $\text{fail}$ 指针，于是我们跳到 $\text{fail}[5]=10$ 发现没有 $s$ 连出的字典树的边，于是跳到 $\text{fail}[10]=0$，发现有 $\text{trie}[0][s]=7$，于是 $\text{fail}[6]=7$；但是有了黑边、蓝边，我们跳到 $\text{fail}[5]=10$ 之后直接走 $trans(10,s)=7$ 就走到 $7$ 号节点了。

多模式匹配

int query(char *t)
{
    int p=0,ans=0;
    int len=strlen(t);
    for(int i=0;i<len;i++)
    {
        int ch=t[i]-'a';
        p=trie[p][ch];
        for(int j=p;j&&ed[j]!=-1;j=fail[j])
        {
            ans=ans+ed[j];
            ed[j]=-1;
        }
    }
    return ans;
}

  定义 $p$ 为字典树上当前匹配到的节点，$ans$ 即返回的答案。循环遍历匹配串，$p$ 在字典树上跟踪当前字符。利用 $\text{fail}$ 指针找出所有匹配的模式串，累加到答案中，然后清 $0$。对 $ed[j]$ 取反的操作用来判断 $ed[j]$ 是否等于 $-1$。在上文中我们分析过，字典树的结构其实就是一个 $trans$ 函数，构建好这个函数后，在匹配字符串的过程中，我们会舍弃部分前缀达到最低限度的匹配。$\text{fail}$ 指针则指向了更多的匹配状态。

  对于刚才的自动机，我们从根节点开始尝试匹配 $\text{ushersheishis}$，那么 $p$ 的变化将是：

  $1.$红色节点：$p$ 节点。

  $2.$粉色箭头：$p$ 在自动机上的跳转。

  $3.$蓝色的边：成功匹配的模式串。

  $4.$蓝色节点：跳 $\text{fail}$ 指针时的节点（状态）。

时间复杂度分析

  $\text{AC}$自动机的时间复杂度在需要找到所有匹配位置时是 $O(|s|+m)$，其中 $|s|$ 表示文本串的长度，$m$ 表示模板串的总匹配次数；而只需要求是否匹配的时间复杂度为 $O(|s|)$。

P3808 【模板】AC自动机（简单版）

  给定 $n$ 个模式串和 $1$ 个文本串，求有多少个模式串在文本串里出现过（模式串长度之和与文本串长度均 $\leq 10^6$）。

#include<bits/stdc++.h>
using namespace std;
const int N=1000010;
int trie[N][26],tot,n;
int ed[N],fail[N];
char s[N];
void insert(char *str)
{
    int len=strlen(str),p=0;
    for(int i=0;i<len;i++)
    {
        int ch=str[i]-'a';
        if(trie[p][ch]==0)
            trie[p][ch]=++tot;
        p=trie[p][ch];
    }
    ed[p]++;
}
queue<int> Q;
void build()
{
    for(int i=0;i<26;i++)
        if(trie[0][i]!=0)
            Q.push(trie[0][i]);
    while(!Q.empty())
    {
        int u=Q.front();
        Q.pop();
        for(int i=0;i<26;i++)
        {
            if(trie[u][i]!=0)
            {
                fail[trie[u][i]]=trie[fail[u]][i];
                Q.push(trie[u][i]);
            }
            else
                trie[u][i]=trie[fail[u]][i];
        }
    }
}
int query(char *t)
{
    int p=0,ans=0;
    int len=strlen(t);
    for(int i=0;i<len;i++)
    {
        int ch=t[i]-'a';
        p=trie[p][ch];
        for(int j=p;j&&ed[j]!=-1;j=fail[j])
        {
            ans=ans+ed[j];
            ed[j]=-1;
        }
    }
    return ans;
}
int main()
{
    cin>>n;
    for(int i=1;i<=n;i++)
    {
        scanf("%s",s);
        insert(s);
    }
    scanf("%s",s);
    build();
    printf("%d",query(s));
    return 0;
}

P3796 【模板】AC自动机（加强版）

题目描述

  有 $N$ 个由小写字母组成的模式串以及一个文本串 $T$。每个模式串可能会在文本串中出现多次。需要找出哪些模式串在文本串 $T$ 中出现的次数最多。

输入格式

  输入含多组数据。

  每组数据的第一行为一个正整数 $N$，表示共有 $N$ 个模式串（$1 \leq N \leq 150$）。

  接下去 $N$ 行，每行一个长度小于等于 $70$ 的模式串。下一行是一个长度小于等于 $10^6$ 的文本串 $T$。

  输入结束标志为 $N=0$。

输出格式

  对于每组数据，第一行输出模式串最多出现的次数，接下去若干行每行输出一个出现次数最多的模式串，按输入顺序排列。

代码

#include<bits/stdc++.h>
using namespace std;
const int N=160*80;
int tot,trie[N][26],fail[N],ed[N],val[N],cnt[N]; 
char s[N][100],t[1000010];
void init()
{
    memset(fail,0,sizeof(fail));//fail指针 
    memset(trie,0,sizeof(trie));//字典树
    memset(val,0,sizeof(val));//每个字符串出现的次数
    memset(cnt,0,sizeof(cnt));//下标为i的字符串出现的次数
    memset(ed,0,sizeof(ed));//以p为结尾的字符串的下标 
    tot=0; 
}
void insert(char *s,int ID)
{
    int p=0;
    int len=strlen(s);
    for(int i=0;i<len;i++)
    {
        int ch=s[i]-'a';
        if(trie[p][ch]==0)
            trie[p][ch]=++tot;
        p=trie[p][ch];
    }
    ed[p]=ID;
}
queue<int> Q;
void build()
{
    for(int i=0;i<26;i++)
        if(trie[0][i])
            Q.push(trie[0][i]);
    while(!Q.empty())
    {
        int u=Q.front();
        Q.pop();
        for(int i=0;i<26;i++)
        {
            if(trie[u][i])
            {
                fail[trie[u][i]]=trie[fail[u]][i];
                Q.push(trie[u][i]);
            }
            else
                trie[u][i]=trie[fail[u]][i];
        }
    }
}
int query(char *t)//返回最大的出现次数 
{
    int p=0,ans=0;
    int len=strlen(t);
    for(int i=0;i<len;i++)
    {
        int ch=t[i]-'a';
        p=trie[p][ch];
        for(int j=p;j;j=fail[j])
            val[j]++;
    }
    for(int i=0;i<=tot;i++)
    {
        if(ed[i]!=0)
        {
            ans=max(ans,val[i]);
            cnt[ed[i]]=val[i];
        }
    }
    return ans;
}
int main()
{
    int n;
    while(~scanf("%d",&n))
    {
        if(n==0)
            break;
        init();
        for(int i=1;i<=n;i++)
        {
            scanf("%s",s[i]);
            insert(s[i],i);
        }
        build();
        scanf("%s",t);
        int ans=query(t);
        printf("%d\n",ans);
        for(int i=1;i<=n;i++)
            if(cnt[i]==ans)
                printf("%s\n",s[i]);    
    }
    return 0;
}
