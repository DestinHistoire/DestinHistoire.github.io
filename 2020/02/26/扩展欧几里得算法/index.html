扩展欧几里得算法

贝祖定理

  对于任意整数 $ a,b $ ，存在一对整数 $ x,y $ ，满足 $ax+by=\gcd(a,b)$。

证明

  在欧几里得算法的最后一步，即 $b=0$ 时，显然有一对整数 $ x=1,y=0 $，使得 $ a\ast 1+0\ast 0=\gcd(a,0) $。

  若 $ b>0 $ ，则 $\gcd(a,b)=\gcd(b,a%b)$。假设存在一对整数 $ x,y $ ，满足 $b\ast x+(a% b)\ast y=\gcd(b,a% b) $。

  $\because b\ast x+(a% b)\ast y=b\ast x+(a-b\lfloor \frac{a}{b} \rfloor)\ast y=a\ast y+b\ast(x-\lfloor\frac{a}{b} \rfloor y)$

  $\therefore$ 令 $ x’=y,y’=x-\lfloor\frac{a}{b} \rfloor y $ ，得到 $ ax’+by’=\gcd(a,b) $。

  对欧几里得算法的递归过程应用数学归纳法，可知贝祖定理成立。

  贝祖定理是按照欧几里得算法的思路证明的，且上述证明同时给出了整数 $x$ 和 $y$ 的计算方法。这种计算方法被称为扩展欧几里得算法。 

int exgcd(int a,int b,int &x,int &y)
{
    if(b==0)
    {
        x=1;
        y=0;
        return a;
    }
    int gcd=exgcd(b,a%b,x,y);
    int temp=x;x=y;y=temp-y*(a/b);
    return gcd;
}

  定义变量 $ \gcd,x_0,y_0 $ ，调用 $ \gcd=\text{exgcd}(a,b,x_0,y_0) $。注意在上述代码中，$x_0,y_0$ 是以引用的方式传递的。上述程序求出方程 $ ax+by=\gcd(a,b) $ 的一组特解 $ x_0,y_0 $，并返回 $ a,b $ 的最大公约数 $\gcd(a,b)$。

  对于更为一般的方程 $ax+by=c$ ，当且仅当 $\gcd(a,b)\mid c $，该方程有解。我们可以先求出 $ ax+by= \gcd(a,b) $ 的一组特解 $ x_0,y_0 $，然后令 $x_0,y_0$ 同时乘上 $ \frac{c}{\gcd(a,b)} $，就得到了 $ax+by=c$ 的一组特解 $(c/d)x_0,(c/d)y_0$。

  事实上，方程 $ax+by=c$ 的通解可以表示为：
$$
x=\frac{c}{d}x_0+k\frac{b}{d},y=\frac{c}{d}y_0-k\frac{a}{d}(k\in Z)
$$
  其中，$k$ 取遍整数集合，$ d=\gcd(a,b) $，$ x_0,y_0 $ 是 $ ax+by=\gcd(a,b) $ 的一组特解。

【例题】POJ-1061 青蛙的约会

题目描述

  两只青蛙在网上相识了，它们聊得很开心，于是觉得很有必要见一面。它们很高兴地发现它们住在同一条纬度线上，于是它们约定各自朝西跳，直到碰面为止。可是它们出发之前忘记了一件很重要的事情，既没有问清楚对方的特征，也没有约定见面的具体位置。不过青蛙们都是很乐观的，它们觉得只要一直朝着某个方向跳下去，总能碰到对方的。但是除非这两只青蛙在同一时间跳到同一点上，不然是永远都不可能碰面的。为了帮助这两只乐观的青蛙，你被要求写一个程序来判断这两只青蛙是否能够碰面，会在什么时候碰面。

  我们把这两只青蛙分别叫做青蛙 $A$ 和青蛙 $B$ ，并且规定纬度线上东经 $0$ 度处为原点，由东往西为正方向，单位长度 $1$ 米，这样我们就得到了一条首尾相接的数轴。设青蛙 $A$ 的出发点坐标是 $x$ ，青蛙 $B$ 的出发点坐标是 $y$。青蛙 $A$一次能跳 $m$ 米，青蛙 $B$ 一次能跳 $n$ 米，两只青蛙跳一次所花费的时间相同。纬度线总长 $L$ 米。现在要你求出它们跳了几次以后才会碰面。

输入格式

  输入只包括一行 $5$ 个整数 $x,y,m,n,L$，其中 $x≠y < 2* 10^9,0 < m,n < 2* 10^9,0 < L < 2.1* 10^9$。

输出格式

  输出碰面所需要的跳跃次数，如果永远不可能碰面则输出一行 $\text{“Impossible”}$。

分析

  两个青蛙跳到同一个点上才算是遇到了，可推出如下式子:

  $(x+m* t)-(y+n* t)=p* L$

  其中：$t$ 是跳的次数，$p$ 是 $A$ 青蛙跳的圈数与$B$青蛙跳的圈数之差。整个就是路程差等于纬度线周长的整数倍。

  转化一下：$(n-m)* t+L* p=x-y$

  令：$ a=n-m,b=L,c=x-y,d=\gcd(a,b) $

  有：$ a\ast t+b\ast p=c $

  要求的是：$ t $ 的最小整数解。

  用扩展欧几里德求出其中一组解 $ t_0,p_0 $，并令：$ d=\gcd(a,b)······(1)$

  有：$ a\ast t_0+b\ast p_0=d······(2) $

  因为：$ d=\gcd(a,b) $，所以 $ a\ast t/d $ 是整数，$ b\ast t/d $ 也是整数，所以 $ c/d $ 也需要是整数，否则无解。   $(2)$ 式两边都乘 $ (c/d) $ ，得到：$ a\ast t_0\ast (c/d)+b\ast p_0\ast (c/d)=c $

  所以：$ t_0\ast (c/d) $ 是最小的解，但有可能是负数。

  因为：$a\ast (t_0\ast(c/d)+b\ast n)+b\ast(
p_0\ast (c/d)-a\ast n)=d $，其中的 $ n $ 是自然数。

  所以，解为 $ (t_0\ast (c/d)%b+b)%b $。

代码

#include<iostream>
using namespace std;
int exgcd(int a,int b,int &x,int &y)
{
    if(b==0)
    {
        x=1;y=0;
        return a;
    }
    int gcd=exgcd(b,a%b,x,y);
    int temp=x;x=y;y=temp-a/b*y;
    return gcd;
}
int main()
{
    int x,y,m,n,l,X,Y,gcd;
    cin>>x>>y>>m>>n>>l;
    gcd=exgcd(n-m,l,X,Y);
    if((x-y)%gcd!=0)
        cout<<"Impossible"<<endl;
    else
        cout<<((long long)(x-y)/gcd*X%(l/gcd)+(l/gcd))%(l/gcd)<<endl;
    return 0;
}

线性同余方程

  给定整数 $a,b,m$，求一个整数 $x$ 满足 $ a\ast x\equiv b(\text{mod}\ m) $，或者给出无解。因为未知数的指数为 $1$ ，所以我们称之为一次同余方程，也称线性同余方程。

  $ a\ast x\equiv b(\text{mod}\ m) $ 等价于 $ a\ast x-b $ 是 $ m $ 的倍数，不妨设为 $ -y $ 倍。于是，该方程可以改写为 $ a\ast x+m\ast y=b $。

  根据贝祖定理及其证明过程，线性同余方程有解当且仅当 $ \gcd(a,m)\mid b $ 。

  在有解时，先用欧几里得算法求出一组整数 $x_0,y_0$，满足 $ a\ast x_0+m\ast y_0=\gcd(a,m) $。然后 $x=x_0* \frac{b}{\gcd(a,m)}$就是原线性同余方程的一个解。

  方程的通解则是所有模 $ \frac{m}{\gcd(a,m)} $ 与 $x$ 同余的整数。

【例题】同余方程 (NOIP2012/CH3301)

  求关于 $x$ 的同余方程 $ a\ast x\equiv 1(\text{mod}\ b)$ 的最小正整数解。 输入数据保证一定有解。

分析

  由上述线性同余方程的知识可得，$ a\ast x\equiv 1(\text{mod}\ b)$ 有解当且仅当 $ \gcd(a,b)=1 $。

  方程可改写为 $ a\ast x+b\ast y=1 $，用欧几里得算法求出一组特解 $ x_0,y_0 $，则 $ x_0 $ 就是原方程的一个解，通解为所有模 $ b $ 与 $ x_0 $ 同余的整数。通过取模操作把解的范围移动到 $ 1 $ ~ $ b $ 之间，就得到了最小正整数解。

代码

#include<bits/stdc++.h>
using namespace std;
long long a,b,x,y;
long long exgcd(long long a,long long b,long long &x,long long &y)
{
    if(b==0)
    {
        x=1;
        y=0;
        return a;
    }
    long long gcd=exgcd(b,a%b,x,y);
    long long temp=x;x=y;y=temp-y*(a/b);
    return gcd;
}
int main()
{
    cin>>a>>b;
    exgcd(a,b,x,y);
    cout<<(x%b+b)%b<<endl;
    return 0;
}
